# 班级批量导入功能设计文档

## 1. 功能概述

### 1.1 功能目标
提供班级批量导入功能，支持通过 Excel 文件批量创建班级，减少手动创建的工作量。

### 1.2 应用场景
- **场景一：县区管理端批量导入**
  - 用户：区县管理员、系统管理员
  - 权限：可以导入任意学校的班级
  - 特点：需要指定学校信息
  
- **场景二：学校端批量导入**
  - 用户：学校管理员
  - 权限：只能导入本校的班级
  - 特点：自动使用当前用户的学校，无需指定学校信息

> - ✅ 消除数据冗余（避免 Classroom.head_teacher_id 和 ClassroomMembership 同时存储）
> - ✅ 统一管理角色关系（所有班级成员关系都在 ClassroomMembership 中）
> - ✅ 更灵活（支持一个班级有多个班主任角色的成员）
> - ✅ 简化导入流程（导入时教师可能还未创建，避免匹配失败）
>
> 导入完成后，管理员可以通过"班级成员管理"功能添加班主任。

## 3. 数据验证规则

### 3.1 必需字段验证
- 学校端：年级级别、班级编号
- 县区端：学校名称、年级级别、班级编号

### 3.2 学校验证（仅县区端）
- 优先通过学校代码精确匹配
- 其次通过学校名称+区域匹配
- 如果找不到学校，返回错误

### 3.3 年级验证
- 通过年级级别（level）精确匹配（必填字段）
- 如果找不到对应的年级，返回错误
- 如果用户提供了年级名称，验证是否与数据库中的年级名称匹配（可选验证）
- 如果用户未提供年级名称，使用数据库中年级对象的name字段

### 3.4 班级编号验证
- 验证班级编号格式（如：501 = 5年级1班，1001 = 10年级1班）
- 解析班级编号，提取年级级别和班级序号
- 验证年级级别范围（1-12）
- 验证班级序号范围（1-99）

### 3.5 班级名称生成
- 如果未提供班级名称，根据班级编号和年级名称自动生成
- 生成规则：`{年级名称}{班级序号}班`
- 例如：班级编号 `701` + 年级名称 `七年级` → `七年级1班`

### 3.6 班级唯一性验证
- 检查学校+年级+班级编号的唯一性（优先）
- 检查学校+年级+班级名称的唯一性（如果提供了班级名称）
- 如果班级编号已存在，检查是否属于同一学校+年级
- 如果已存在相同班级，可选择更新或跳过

### 3.7 数据格式验证
- 入学年份：必须是有效的年份（1900-2100），可在导入界面统一设置
- 班级容量：必须是正整数（1-200），可在导入界面统一设置
- 班级编号：字符串格式（如：501、1001），长度不超过50

### 3.8 统一设置优先级
- 如果导入API提供了`enrollment_year`或`capacity`参数，所有班级使用参数值
- Excel中的入学年份和班级容量字段将被忽略（如果提供了统一设置）
- 如果未提供统一设置，则使用Excel中的值（如果填写了）

## 4. 数据匹配策略

### 4.1 学校匹配（仅县区端）
```python
# 优先级1：学校代码精确匹配
if school_code:
    school = find_school_by_code(school_code)

# 优先级2：学校名称+区域匹配
if not school:
    school = find_school_by_name_and_region(school_name, region_id)
```

### 4.2 年级匹配
```python
# 通过年级级别（level）精确匹配（必填）
grade = find_grade_by_level(grade_level)

# 如果用户提供了年级名称，验证是否匹配
if grade_name and grade:
    if grade.name != grade_name:
        # 警告：用户提供的年级名称与数据库中的不匹配
        logger.warning(f"年级级别 {grade_level} 对应的年级名称应为 '{grade.name}'，但提供了 '{grade_name}'")
```

## 5. 导入逻辑流程

### 5.1 解析阶段
1. 验证文件格式（.xlsx, .xls）
2. 读取表头，匹配列名
3. 验证必需列是否存在
4. 逐行解析数据，记录解析错误

### 5.2 验证阶段
对每条记录进行验证：
1. 验证学校（县区端）
2. 验证年级（通过年级级别查找）
3. 获取年级名称（从数据库获取，如果用户提供了则验证匹配）
4. 验证班级编号格式（解析年级级别和班级序号）
5. 生成班级名称（如果未提供）
6. 验证班级唯一性
7. 验证可选字段格式

### 5.3 导入阶段
1. 查找或匹配学校、年级
2. 通过年级级别查找年级对象，获取年级名称
3. 解析班级编号，生成班级名称（如果未提供）
4. 应用统一设置（如果提供了enrollment_year和capacity参数，覆盖Excel中的值）
5. 检查班级是否已存在（通过班级编号或班级名称）
6. 创建新班级或更新现有班级
7. 记录导入结果（创建/更新/跳过/失败）

**年级名称获取逻辑：**
```python
# 通过年级级别查找年级对象（必填）
grade = find_grade_by_level(grade_level)  # level=7 -> Grade(name="七年级")

# 使用数据库中的年级名称
grade_name = grade.name  # "七年级"
```

**班级名称生成逻辑：**
```python
# 解析班级编号
grade_level, class_seq = parse_classroom_code(classroom_code)  # 701 -> (7, 1)

# 获取年级对象（已通过年级级别获取）
grade = find_grade_by_level(grade_level)  # level=7 -> Grade(name="七年级")

# 生成班级名称（如果用户未提供）
if not classroom_name:
    classroom_name = f"{grade.name}{class_seq}班"  # "七年级1班"
```

### 5.4 结果返回
返回导入统计：
- 总记录数
- 成功数（创建+更新）
- 失败数
- 创建数
- 更新数
- 跳过数
- 详细错误列表

## 6. API 设计

### 6.1 县区管理端 API

#### 6.1.1 导入班级
```
POST /api/v1/admin/organization/classrooms/import
```

**请求参数：**
- `file`: File（Excel文件）
- `update_existing`: bool = False（是否更新已存在的班级）
- `enrollment_year`: Optional[int] = None（统一设置入学年份，如果提供，所有班级使用此值；Excel中的值将被忽略）
- `capacity`: Optional[int] = None（统一设置班级容量，如果提供，所有班级使用此值；Excel中的值将被忽略）

**权限：**
- 系统管理员、区县管理员：可以导入任意学校的班级（Excel中需包含学校信息）
- 学校管理员：只能导入本校的班级（自动使用 `current_user.school_id`）

**响应：**
```json
{
  "total": 50,
  "success": 48,
  "failed": 2,
  "created": 45,
  "updated": 3,
  "skipped": 0,
  "errors": [
    {
      "row": 5,
      "field": "年级级别",
      "message": "年级级别 13 不存在"
    }
  ]
}
```

**Schema定义（在 `backend/app/api/v1/admin_organization.py` 中）：**
```python
class ClassroomImportError(BaseModel):
    """班级导入错误"""
    row: int = Field(..., description="行号")
    field: Optional[str] = Field(None, description="字段名")
    message: str = Field(..., description="错误信息")

class ClassroomImportResponse(BaseModel):
    """班级批量导入响应"""
    total: int = Field(..., description="总记录数")
    success: int = Field(..., description="成功数")
    failed: int = Field(..., description="失败数")
    created: int = Field(0, description="创建的班级数")
    updated: int = Field(0, description="更新的班级数")
    skipped: int = Field(0, description="跳过的班级数（已存在）")
    errors: List[ClassroomImportError] = Field(default_factory=list, description="错误列表")
```

#### 6.1.2 下载模板
```
GET /api/v1/admin/organization/classrooms/import/template
```

**查询参数：**
- `school_admin`: bool = False（如果为true，返回不包含学校字段的简化模板）

返回 Excel 模板文件：
- 默认：返回县区管理端模板（包含学校字段）
- `school_admin=true`：返回学校管理端模板（不包含学校字段）

### 6.2 学校端 API

**注意：** 学校端使用相同的API端点，但通过权限检查确保只能导入本校的班级。

#### 6.2.1 导入班级
```
POST /api/v1/admin/organization/classrooms/import
```

**请求参数：**
- `file`: File（Excel文件）
- `school_id`: Optional[int] = None（学校ID，如果提供且用户是学校管理员，必须是其所属学校；如果不提供，使用current_user.school_id）
- `update_existing`: bool = False（是否更新已存在的班级）
- `enrollment_year`: Optional[int] = None（统一设置入学年份，如果提供，所有班级使用此值；Excel中的值将被忽略）
- `capacity`: Optional[int] = None（统一设置班级容量，如果提供，所有班级使用此值；Excel中的值将被忽略）

**权限检查：**
- 如果用户是学校管理员（`school_admin`），只能导入本校的班级
- API会自动使用 `current_user.school_id`，Excel中不需要学校信息
- 如果用户是区县管理员或系统管理员，可以导入任意学校的班级（需要Excel中包含学校信息）

**响应：**同县区管理端

GET /api/v1/admin/organization/classrooms/import/template?school_admin=true

**查询参数：**

## 8. 服务层设计

### 8.1 ClassroomImportService

**文件位置：** `backend/app/services/classroom_import_service.py`

**异常类：**
```python
class ClassroomImportServiceError(Exception):
    """班级导入服务错误"""
    pass
```

**主要方法：**

1. `parse_classroom_excel(file_path: str, is_school_admin: bool = False) -> Tuple[List[Dict[str, Any]], List[Dict[str, Any]]]`
   - 解析 Excel 文件
   - 区分县区端和学校端的列名（根据 `is_school_admin` 参数）
   - 返回：(记录列表, 错误列表)

2. `parse_classroom_code(classroom_code: str) -> Tuple[Optional[int], Optional[int]]`
   - 解析班级编号，提取年级级别和班级序号
   - 例如：701 -> (7, 1)，1001 -> (10, 1)
   - 可以复用 `StudentImportService.parse_classroom_code` 方法

3. `async find_grade_by_level(db: AsyncSession, grade_level: int) -> Optional[Grade]`
   - 通过年级级别查找年级对象（必填字段）
   - 返回Grade对象，包含年级名称

4. `generate_classroom_name(grade_name: str, class_seq: int) -> str`
   - 根据年级名称和班级序号生成班级名称
   - 例如：("七年级", 1) -> "七年级1班"

5. `async find_school(db: AsyncSession, school_name: str, region_id: Optional[int] = None, school_code: Optional[str] = None) -> Optional[School]`
   - 查找学校（仅县区端需要）
   - 可以复用 `StudentImportService.find_school` 方法

6. ~~`async find_teacher(db: AsyncSession, teacher_name: str, school_id: int) -> Optional[User]`~~（已移除）
   - ~~查找班主任（通过姓名或用户名）~~
   - ~~验证角色是否为教师（`UserRole.TEACHER`）~~
   - **不再需要**：班主任信息通过"班级成员管理"功能添加，不在导入时处理

7. `async import_classrooms(db: AsyncSession, records: List[Dict[str, Any]], school_id: Optional[int] = None, update_existing: bool = False, enrollment_year: Optional[int] = None, capacity: Optional[int] = None) -> Dict[str, Any]`
   - 批量导入班级
   - 支持创建和更新
   - 自动获取年级名称（通过年级级别）
   - 自动生成班级名称（如果未提供）
   - 支持统一设置入学年份和班级容量（如果提供参数，覆盖Excel中的值）
   - 返回结果字典：`{"total": int, "success": int, "failed": int, "created": int, "updated": int, "skipped": int, "errors": List[Dict]}`

## 9. 错误处理

### 9.1 解析错误
- 缺少必需列
- 文件格式错误
- 空文件

### 9.2 验证错误
- 学校不存在（县区端）
- 年级级别不存在（年级级别不在1-12范围内，或数据库中不存在该级别的年级）
- 班级编号格式错误（无法解析）
- 班级编号已存在（且不允许更新）

### 9.3 业务错误
- 班级已存在（且不允许更新）
- 数据完整性错误

## 11. 技术要点

### 11.1 列名映射
支持多种列名格式：
- `年级级别` / `年级级别*` / `级别` / `level`
- `年级名称` / `年级` / `年级名称*`（选填）
- `班级编号` / `班级代码` / `班级编号*` / `班级编码`
- `班级名称` / `班级` / `班级名称*`（选填）
- ~~`正班主任` / `班主任` / `班主任姓名`~~（已移除）

### 11.2 年级级别说明
- 年级级别范围：1-12
- 映射关系（标准）：
  - 1-6：小学（一年级到六年级）
  - 7-9：初中（七年级到九年级，初一、初二、初三）
  - 10-12：高中（高一、高二、高三）
- 年级名称从数据库获取：通过年级级别查找Grade对象，使用其name字段
- 如果用户提供了年级名称，验证是否与数据库中的名称匹配（可选验证）

### 11.3 班级编号解析规则
- 格式：`{年级级别}{班级序号}`
- 示例：
  - `501` = 5年级1班 (grade_level=5, class_seq=1)
  - `701` = 7年级1班 (grade_level=7, class_seq=1)
  - `1001` = 10年级1班 (grade_level=10, class_seq=1) 高一1班
  - `1203` = 12年级3班 (grade_level=12, class_seq=3) 高三3班
- 验证范围：年级级别 1-12，班级序号 1-99

### 11.4 班级名称生成规则
- 如果用户提供了班级名称，直接使用
- 如果未提供，自动生成：`{年级名称}{班级序号}班`
- 示例：
  - 班级编号 `701` + 年级名称 `七年级` → `七年级1班`
  - 班级编号 `1001` + 年级名称 `高一` → `高一1班`

### 11.5 班级唯一性
唯一性约束：
- 通过班级编号：`(school_id, grade_id, code)` 如果提供了 code
- 通过班级名称：`(school_id, grade_id, name)`
- 同一学校、同一年级，班级编号或班级名称必须唯一

### 11.4 更新策略
- `update_existing=False`：如果班级已存在，跳过
- `update_existing=True`：如果班级已存在，更新可选字段

## 12. 注意事项

1. **权限控制**
   - 县区管理员：可以导入任意学校的班级
   - 学校管理员：只能导入本校的班级
   - 系统管理员：拥有全部权限

2. **数据一致性**
   - 导入前检查学校、年级是否存在
   - 导入后验证数据完整性

3. **性能优化**
   - 批量提交（每100条提交一次）
   - 使用缓存减少重复查询

4. **用户体验**
   - 提供详细的错误信息
   - 支持部分成功导入（不会因为部分错误而全部失败）
   - 显示导入进度（可选）
