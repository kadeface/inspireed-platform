# 学生导入模板学校名称字段修复

## 🐛 问题描述

```
导入失败: 缺少必需列: 学校名称*
```

**原因**: 前端模板使用了 `学校代码` 字段，但后端 `student_account_import_strategy.py` 期望 `学校名称*` 作为区管理员模式的必填字段。

---

## 🔍 根本原因

### 后端期望的字段

```python
DISTRICT_COLUMN_MAPPING = {
    "学校名称": "school_name",  # 必填字段
    "年级级别": "grade_level",
    "班级编号": "classroom_code",
    # ...
}

DISTRICT_REQUIRED_COLUMNS = [
    "school_name",      # 必需
    "grade_level",      # 必需
    "classroom_code",   # 必需
    "student_id_number", # 必需
    "full_name"         # 必需
]
```

### 修复前的模板（错误）

```typescript
const template = [
  ['学号*', '姓名*', '学籍号*', '邮箱', '手机号', '性别', '学校代码', '年级级别*', '班级编号*'],
  //                                                              ^^^^^^^^ 错误！
  ['2024100001', '张三', '123456789012345678', 'zhang@example.com', '13800138000', '男', '10001', 7, '701'],
  //                                                                                  ^^^^^ 使用代码
]
```

**问题**:
- 后端期望 `学校名称*`（学校全名）
- 模板提供 `学校代码`（数字编码）
- 后端无法找到 `学校名称` 列，导入失败

## 📝 使用指南

### 1. 下载新模板

进入：**组织架构管理 → 人员管理 → 学生管理 → 批量导入 → 下载导入模板**

新模板格式：

```
| 学号* | 姓名* | 学籍号* | 邮箱 | 手机号 | 性别 | 学校名称* | 年级级别* | 班级编号* |
|-------|-------|---------|------|--------|------|----------|----------|----------|
| 2024100001 | 张三 | 123456789012345678 | zhang@example.com | 13800138000 | 男 | 示例学校 | 7 | 701 |
```

### 2. 填写数据

**必填字段**:
- **学号***: 学生登录用户名
- **姓名***: 学生真实姓名
- **学籍号***: 唯一标识（身份证号等）
- **学校名称***: 学校全名（必须在系统中存在）
- **年级级别***: 数字（1-12）
- **班级编号***: 班级代码（如701、1001）

**可选字段**:
- 邮箱
- 手机号
- 性别

### 3. 导入流程

```
1. 下载最新模板
2. 填写学生数据（使用学校全名）
3. 上传Excel文件
4. 预览数据
5. 点击"开始导入"
6. 查看导入结果
```

## ⚠️ 常见错误

### 1. 学校名称不存在

**错误**: `学校 'XXX' 未找到`

**原因**: 填写的学校名称与系统中注册的学校名称不匹配

**解决**:
- 在"学校管理"中查看准确的学校名称
- 确保名称完全匹配（包括空格、标点）
- 或使用"学校批量导入"先创建学校

### 2. 班级不存在

**错误**: `班级编号 'XXX' 不存在`

**原因**: 学生导入前，对应的班级必须已在系统中创建

**解决**:
- 先在"班级管理"中批量导入班级
- 确认班级编号格式正确（年级级别 + 班级序号）

### 3. 学籍号重复

**错误**: `学籍号 'XXX' 已存在`

**原因**: 该学籍号的学生已存在

**解决**:
- 检查是否已导入过该学生
- 或勾选"更新已存在学生"选项
