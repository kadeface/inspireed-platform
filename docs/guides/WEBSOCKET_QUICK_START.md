# WebSocket 实时同步快速启动指南

> **版本：** v1.0  
> **更新日期：** 2025-01-17

## 快速开始

### 1. 启动后端服务

确保后端服务已启动并支持 WebSocket：

```bash
cd backend
uvicorn app.main:app --host 0.0.0.0 --port 8000 --ws auto --reload
```

### 2. 启动前端服务

```bash
cd frontend
npm run dev
```

### 3. 测试 WebSocket 连接

#### 教师端操作：
1. 登录教师账号
2. 打开一个课程
3. 点击"开始授课"创建课堂会话
4. 切换内容（选择不同的模块）

#### 学生端验证：
1. 登录学生账号
2. 打开同一课程
3. 自动加入课堂会话
4. **查看右上角连接状态指示器**：
   - ✅ 绿色"实时同步" = WebSocket 已连接
   - ⚠️ 灰色"轮询模式" = 降级到轮询
5. 教师切换内容后，学生端应**立即**更新（<100ms）

## 常见问题

### Q1: 连接状态显示"轮询模式"怎么办？

**可能原因：**
1. Token 无效或过期
2. 后端 WebSocket 服务未启动
3. 网络代理阻止 WebSocket
4. CORS 配置问题

**解决方法：**
1. 检查浏览器控制台错误日志
2. 确认后端启动时使用了 `--ws auto` 参数
3. 检查 Token 是否正确存储在 localStorage
4. 尝试刷新页面重新连接

### Q2: 连接成功但内容不同步怎么办？

**检查步骤：**
1. 确认教师端已成功切换内容（查看教师端日志）
2. 确认学生端 WebSocket 仍然连接（查看连接状态指示器）
3. 查看浏览器控制台是否收到 `cell_changed` 消息
4. 查看后端日志是否显示"广播消息"

### Q3: 如何查看 WebSocket 连接详情？

**Chrome DevTools：**
1. 打开开发者工具（F12）
2. 切换到"Network"标签
3. 筛选"WS"（WebSocket）
4. 点击连接查看消息收发记录

### Q4: 心跳机制如何验证？

**浏览器控制台：**
每 30 秒应该看到：
```
📨 收到消息: pong
```

**Chrome DevTools Network：**
在 WebSocket 连接的 Messages 标签中，应该看到周期性的 ping/pong 消息。

## 性能监控

### 观察延迟

**工具：** 浏览器控制台

在教师端切换内容时，记录时间戳：
```javascript
console.log('🎯 教师切换时间:', new Date().toISOString())
```

在学生端收到消息时，记录时间戳（已自动记录）：
```
📨 收到消息: cell_changed { timestamp: "2025-01-17T10:30:45Z" }
```

计算延迟：
```
延迟 = 学生收到时间 - 教师切换时间
```

**目标：** <100ms

### 观察连接数

**后端日志：**
```
📊 会话 123 当前在线: 5 人
```

### 观察消息频率

**后端日志：**
```
📢 广播消息到会话 123（5 人）
```

统计每分钟的广播次数，应该等于教师切换内容的次数。
