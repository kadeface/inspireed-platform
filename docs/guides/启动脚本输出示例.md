# 启动脚本输出示例

## 运行 `./start.sh` 后的输出效果

```bash
🎉 服务启动完成！

📱 访问地址：

   【本机访问】
   前端应用: http://localhost:5173
   后端API: http://localhost:8000
   API文档: http://localhost:8000/docs

   【局域网访问】（其他设备使用这些地址）
   前端应用: http://192.168.1.100:5173
   后端API: http://192.168.1.100:8000
   API文档: http://192.168.1.100:8000/docs

   💡 提示：
   - 确保设备连接到同一局域网
   - 防火墙需允许 5173 和 8000 端口
   - 移动设备可访问: http://192.168.1.100:5173

🔐 测试账号：
   管理员: admin@inspireed.com / admin123
   教师: teacher@inspireed.com / teacher123
   学生: student@inspireed.com / student123
   研究员: researcher@inspireed.com / researcher123

📋 管理命令：
   查看日志: tail -f logs/backend.log 或 tail -f logs/frontend.log
   停止服务: ./stop.sh
   重启服务: ./restart.sh
   网络配置: ./get-network-info.sh

🌐 局域网配置：
   查看配置指南: cat 局域网访问配置说明.md
   详细文档: NETWORK_ACCESS_GUIDE.md

✨ 开始使用 InspireEd 吧！
```

## 说明

- **【本机访问】**: 在运行服务的电脑上使用这些地址
- **【局域网访问】**: 在同一局域网内的其他电脑、手机、平板上使用这些地址
- IP 地址会根据实际网络环境自动检测
- 如果无法检测到 IP 地址，则只显示本机访问地址

## 首次使用建议

1. 运行 `./start.sh` 启动服务
2. 查看输出的局域网访问地址
3. 如需配置或了解详情，运行 `./get-network-info.sh`
4. 从其他设备访问显示的局域网地址即可

## 注意事项

- 局域网地址会在每次启动时自动检测
- 如果 IP 地址改变，重启服务后会显示新的地址
- 首次从其他设备访问时，可能需要配置 CORS（参考配置指南）
