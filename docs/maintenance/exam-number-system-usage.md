# 统一考号系统使用指南

## 概述

统一考号系统支持校级考号和市级考号的自动转换，实现学校级考试和市级考试的统一管理。

## 考号格式

### 校级考号格式
```
格式：{入学年份}{班级编号:02d}{流水号:03d}
示例：20250101
- 2025：入学学年
- 01：班级编号
- 01：班级内流水号
```

### 市级考号格式
```
格式：{年级编码}{学校代码}{班级编号:02d}{流水号:03d}
示例：7783190101
- 7：年级编码（7=高一，8=高二，9=高三）
- 78319：学校代码
- 01：班级编号
- 01：班级内流水号
```

## 考试级别

系统支持三种考试级别：

| 级别 | 代码 | 考号格式 | 说明 |
|------|------|----------|------|
| 学校级 | `school` | 校级考号（20250101） | 学校内部考试，直接使用校级考号 |
| 区县统考 | `district` | 市级考号（7783190101） | 区县统一考试，自动转换为市级考号 |
| 市级考试 | `city` | 市级考号（7783190101） | 市级统一考试，自动转换为市级考号 |

## 年级计算规则

年级按**学年**计算（每年9月1日为新学年开始）：

```python
# 示例1：2025年9月入学的学生
入学学年：2025-2026
考试日期：2026年1月15日  → 属于2025-2026学年  → 高一（年级编码7）
考试日期：2026年9月1日   → 属于2026-2027学年  → 高二（年级编码8）
考试日期：2027年1月15日  → 属于2026-2027学年  → 高二（年级编码8）
```

## 考号转换示例

### 校级 → 市级
```
校级考号：20250101
  ├─ 入学年份：2025
  ├─ 班级：01
  └─ 座位：01

转换为市级考号（2026年1月考试）：
7783190101
  ├─ 年级：7（高一，因为2025年入学，2026年考试）
  ├─ 学校：78319
  ├─ 班级：01
  └─ 座位：01
```

### 市级 → 校级
```
市级考号：7783190101
  ├─ 年级：7（高一）
  ├─ 学校：78319
  ├─ 班级：01
  └─ 座位：01

转换为校级考号（2025年入学）：
20250101
  ├─ 入学年份：2025
  ├─ 班级：01
  └─ 座位：01
```

## 使用方法

### 1. 运行数据库迁移

```bash
cd backend
source venv/bin/activate
alembic upgrade head
```

### 2. 创建考试时指定级别

**前端使用**：
在创建考试表单中选择"考试级别"：
- 学校级考试
- 区县统考
- 市级考试

**API 使用**：
```json
{
  "name": "市模拟考试",
  "exam_type": "mock",
  "exam_level": "city",  // ← 考试级别
  "exam_date": "2026-01-15T00:00:00",
  "semester_id": 1,
  "grade_id": 7
}
```

### 3. 考场编排自动转换

当执行考场编排时：
- **学校级考试**：直接使用学生的校级考号（username）
- **区县/市级考试**：自动转换为市级考号格式

转换逻辑在 `exam_room_service.py` 中自动处理。

### 4. 查看考号

考场编排后，可以在考场管理页面查看：
- 每个考场的考号范围（`exam_number_start` ~ `exam_number_end`）
- 每个学生的具体考号

## 数据字段说明

### users 表
| 字段 | 说明 | 示例 |
|------|------|------|
| `username` | 校级考号 | 20250101 |
| `student_id_number` | 学籍号（唯一追踪） | 440804201111111814 |

### exam_number_mappings 表
| 字段 | 说明 | 示例 |
|------|------|------|
| `exam_number` | 考号（根据考试级别） | 20250101 或 7783190101 |
| `student_id_number` | 学籍号（冗余） | 440804201111111814 |

### exam_room_students 表
| 字段 | 说明 | 示例 |
|------|------|------|
| `exam_number` | 考号（根据考试级别） | 20250101 或 7783190101 |

## 设计优势

1. **自动转换**：考场编排时自动根据考试级别选择考号格式
2. **统一管理**：一个系统支持学校级和市级考试
3. **可追溯性**：学籍号作为唯一标识，跨考试追踪学生表现
4. **灵活性**：支持手动导入市级下发的考号（后续功能）

## 注意事项

1. **学校代码**：确保 `schools` 表中每所学校都有 `code` 字段
2. **入学年份**：学生导入时需要设置正确的入学年份
3. **考试日期**：市级考试的考试日期会影响年级计算
4. **学籍号**：学籍号必须唯一且正确，用于跨考试追踪
