# 导播台问题排查指南

## 问题：看不到导播台

### 检查步骤

1. **确认课堂控制面板已打开**
   - 进入教案编辑页面
   - 切换到"预览模式"（右上角按钮）
   - 确保教案状态为"已发布"
   - 点击工具栏的"课堂控制"按钮（绿色按钮）
   - 应该能看到"教师控制面板"

2. **检查教案是否有内容**
   - 导播台需要教案有至少一个模块（Cell）
   - 如果教案内容为空，导播台不会显示
   - 在编辑模式下添加一些模块后再查看

3. **检查浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签页是否有错误信息
   - 查看 Network 标签页，确认 API 请求是否成功

4. **检查组件是否正确加载**
   - 在控制台输入：`document.querySelector('.classroom-control-board')`
   - 如果返回 `null`，说明组件没有渲染
   - 如果返回元素，说明组件已渲染但可能被隐藏

### 显示条件

导播台会在以下情况下显示：

```javascript
// 显示条件
lesson && lesson.content && lesson.content.length > 0
```

### 两种显示模式

1. **预览模式**（未开始上课）
   - 显示所有模块的链条
   - 节点不可点击（只读）
   - 显示提示："请先开始上课以使用导播台"

2. **活动模式**（已开始上课）
   - 显示所有模块的链条
   - 节点可点击切换
   - 当前激活的节点高亮显示
   - 支持活动控制

### 常见问题

#### Q: 为什么导播台是灰色的？
A: 这是预览模式，表示还没有开始上课。点击"开始上课"按钮后，导播台会变为可交互状态。

#### Q: 点击节点没有反应？
A: 确保已经点击"开始上课"按钮创建了课堂会话。只有在上课状态下才能切换模块。

#### Q: 导播台显示"请先开始上课"？
A: 这是正常的。导播台需要课堂会话才能正常工作。点击"▶️ 开始上课"按钮即可。

### 调试信息

如果仍然看不到导播台，请检查：

1. **浏览器控制台错误**
   ```javascript
   // 在控制台执行
   console.log('Lesson:', window.currentLesson)
   console.log('Session:', window.currentSession)
   ```

2. **组件渲染状态**
   ```javascript
   // 检查组件是否存在
   const panel = document.querySelector('.teacher-control-panel')
   const board = document.querySelector('.classroom-control-board')
   console.log('Panel:', panel)
   console.log('Board:', board)
   ```

3. **Vue 组件状态**
   - 打开 Vue DevTools（如果已安装）
   - 查找 `TeacherControlPanel` 组件
   - 检查 `lesson` 和 `session` 的值

### 解决方案

如果以上检查都正常，但仍然看不到导播台：

1. **刷新页面**
   - 按 Ctrl+F5（Windows）或 Cmd+Shift+R（Mac）强制刷新

2. **清除浏览器缓存**
   - 清除浏览器缓存和 Cookie
   - 重新登录

3. **检查网络连接**
   - 确认前端服务正在运行
   - 检查 API 请求是否成功

4. **查看日志**
   - 检查后端日志是否有错误
   - 检查前端控制台是否有错误
