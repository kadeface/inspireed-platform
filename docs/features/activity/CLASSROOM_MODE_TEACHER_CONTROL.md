# 课堂模式 - 教师内容控制功能

## 功能概述

课堂模式下的内容控制功能，允许教师在上课过程中手动控制学生端显示的内容。这更符合实际教学流程：

1. **默认隐藏**：开始上课时，默认所有模块都不对学生显示
2. **手动切换**：教师根据教学进度，手动切换到需要显示的模块
3. **实时同步**：学生端实时跟随教师的切换，只显示教师指定的模块

## 教师端操作

### 1. 开始上课

1. 点击"▶️ 开始上课"按钮
2. 系统创建会话，**默认不显示任何内容**（`current_cell_id = null`）
3. 学生端会显示"等待教师切换内容"的提示

### 2. 切换内容

教师可以通过以下方式切换内容：

#### 方式1：使用下拉菜单
- 在"内容控制"面板中，使用下拉菜单选择要显示的模块
- 选择"-- 隐藏所有内容 --"可以隐藏所有内容
- 选择具体模块后，学生端会立即显示该模块

#### 方式2：使用导航按钮
- 点击"← 上一个"切换到上一个模块
- 点击"下一个 →"切换到下一个模块

#### 方式3：快速操作按钮
- **"🚫 隐藏所有内容"**：隐藏当前显示的所有内容
- **"▶️ 显示第一个模块"**：快速切换到第一个模块

### 3. 当前状态指示

- **✓ 学生可见**：绿色标识，表示当前模块对学生可见
- **⚠️ 学生不可见**：橙色标识，表示当前没有显示任何内容给学生

### 4. 活动控制

当切换到活动类型的模块时：
- 可以点击"🎯 开始活动"按钮开始活动
- 学生端会显示活动内容，学生可以开始答题
- 教师可以查看学生的提交情况
- 点击"✅ 结束活动"结束活动

## 学生端体验

### 1. 加入课堂

- 学生打开对应的教案页面
- 如果检测到有活跃的课堂会话，会自动加入
- 页面顶部显示"🎓 正在上课"的横幅

### 2. 等待内容

- 如果教师还未切换到任何模块，学生端显示：
  - "等待教师切换内容"的提示
  - "教师正在准备课程内容，请稍候..."
  - 学生无法看到任何课程内容

### 3. 查看内容

- 教师切换模块后，学生端立即显示相应的模块
- 学生只能看到教师当前指定的模块
- 其他模块对学生不可见

### 4. 活动参与

- 当教师开始活动时，学生端会显示活动内容
- 学生可以开始答题
- 提交的答案会实时同步到教师端

## 技术实现

### 后端

1. **会话创建**：
   - 默认设置：`sync_mode: "strict"`（严格同步）
   - 初始状态：`current_cell_id: null`（不显示任何内容）

2. **内容切换API**：
   - `POST /api/v1/classroom-sessions/sessions/{session_id}/navigate`
   - 支持 `cell_id: 0` 来隐藏所有内容（设置 `current_cell_id = null`）
   - 支持 `cell_id: <数字>` 来显示指定的Cell

3. **会话状态**：
   - `current_cell_id`: 当前显示的Cell ID（null表示隐藏所有内容）
   - `settings.sync_mode`: 同步模式（"strict"表示严格同步）

### 前端

1. **学生端**：
   - `useClassroomSession` composable 管理会话状态
   - `filteredCells` 计算属性过滤显示的Cell
   - 如果 `current_cell_id` 为 null，显示"等待教师切换内容"提示

2. **教师端**：
   - `TeacherControlPanel` 组件提供内容控制界面
   - 下拉菜单、导航按钮、快速操作按钮
   - 实时显示当前状态（可见/不可见）

## 使用场景

### 场景1：逐步讲解
1. 教师开始上课，默认不显示内容
2. 教师讲解背景知识
3. 教师切换到第一个模块，学生看到相应内容
4. 教师继续讲解，切换到下一个模块
5. 逐步展示所有内容

### 场景2：重点讲解
1. 教师开始上课
2. 教师直接切换到重点模块
3. 学生只看到重点内容
4. 教师可以随时隐藏内容，回到讲解状态

### 场景3：活动教学
1. 教师切换到活动模块
2. 教师点击"开始活动"
3. 学生开始答题
4. 教师查看学生提交情况
5. 教师结束活动，切换回讲解模式

## 注意事项

1. **Cell ID 问题**：
   - 如果Cell使用UUID（前端临时ID），需要先保存到数据库
   - 保存后，Cell会获得数字ID，才能正常切换
   - 建议：在编辑模式下保存教案，确保所有Cell都有数据库ID

2. **同步延迟**：
   - 当前使用轮询方式同步，可能有轻微延迟
   - 未来可以通过WebSocket实现实时同步

3. **权限控制**：
   - 只有创建会话的教师可以切换内容
   - 学生无法自行切换内容

4. **会话状态**：
   - 只有在"上课中"（active）状态下才能切换内容
   - 暂停或结束后无法切换

## 未来改进

1. **WebSocket实时同步**：实现真正的实时同步，减少延迟
2. **批量切换**：支持同时显示多个模块
3. **预设路径**：支持预设教学路径，自动切换
4. **学生反馈**：学生可以请求查看某个模块（需要教师批准）
5. **进度跟踪**：记录教师切换历史，便于复盘
