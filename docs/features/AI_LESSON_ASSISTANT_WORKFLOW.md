# AI 教案助理工作流程详解

## 概述

AI 教案助理（`LessonAiAssistantDrawer`）是一个帮助教师优化教案的智能工具。当教师选择一个教案后，该工具可以基于教案的结构和内容，提供教学目标、活动设计、优化建议等智能推荐。

## 完整工作流程

### 1. 用户触发阶段

**位置**: `frontend/src/pages/Teacher/LessonEditor.vue`

当教师在教案编辑器中打开一个教案时：

1. **教案数据加载**
   - 从 `lessonStore` 获取当前教案数据（`currentLesson`）
   - 提取教案标题（`lessonTitle`）
   - 计算教案结构概览（`lessonOutline`）

2. **打开 AI 助理抽屉**
   - 用户点击触发按钮，设置 `showLessonAssistant = true`
   - `LessonAiAssistantDrawer` 组件被激活

### 2. 教案结构分析阶段

**位置**: `frontend/src/pages/Teacher/LessonEditor.vue` (第1144-1155行)

```typescript
const lessonOutline = computed(() => {
  if (!currentLesson.value || !Array.isArray(currentLesson.value.content)) {
    return ''
  }

  const items = currentLesson.value.content
    .slice(0, 6)  // 只取前6个单元
    .map((cell, index) => summarizeCell(cell as Cell, index))
    .filter((item): item is string => Boolean(item))

  return items.join('\n')
})
```

**`summarizeCell` 函数** (第2023-2053行) 会分析每个教案单元（Cell）：
- 识别单元类型（文本、代码、视频、活动、流程图等）
- 提取关键信息：
  - 文本单元：提取前28个字符的纯文本
  - 活动单元：提取活动标题
  - 视频单元：提取视频标题
  - 其他类型：使用类型标签

**输出示例**:
```
第1单元：文本 - 课程导入：介绍智能农业的基本概念…
第2单元：视频 - 智能农业技术应用
第3单元：活动 - 设计智能灌溉系统
第4单元：代码 - Python 传感器数据采集
```

### 3. 组件初始化阶段

**位置**: `frontend/src/components/Teacher/LessonAiAssistantDrawer.vue`

组件接收以下 props：
- `lessonTitle`: 教案标题
- `lessonOutline`: 教案结构概览（由步骤2生成）

组件内部状态：
- `activeTab`: 当前标签页（'overview' 或 'resources'）
- `question`: 用户输入的问题
- `response`: AI 返回的回答
- `isSubmitting`: 提交状态

### 4. 智能推荐生成阶段

**预设提示词** (第305-340行):

组件会根据教案信息生成三类提示词：

**基础提示词**（始终可用）:
1. `请根据"${lessonTitle}"生成教学目标和达成指标。`
2. `帮我设计一个课堂导入活动，强调学生参与感。`
3. `为本课安排三个课堂活动，并给出时间分配建议。`
4. `请提供学习评价/作业设计，突出形成性反馈。`
5. `给出课后延伸或家校交流建议，支持学生持续实践。`

**结构相关提示词**（当有 `lessonOutline` 时）:
1. `结合当前教案结构，优化活动衔接和过渡语。`
2. `根据教案单元，提炼每个单元的核心提示语。`
3. `请指出教案结构中的薄弱环节，并给出调整方案。`

**学习科学提示词**（新增，基于学习科学理论）:
1. `基于布鲁姆分类法，为本课设计从记忆到创造的认知层次活动。`
2. `设计苏格拉底式提问序列，引导学生逐步深入思考。`
3. `结合费曼学习法，设计让学生用自己的话解释概念的活动。`
4. `分析不同学习风格学生的特点，提供差异化教学建议。`
5. `基于5E教学模型，优化本课的探索和解释环节。`
6. `设计促进学生主动输出的活动，培养元认知能力。`
7. `结合最近发展区理论，为本课设计脚手架支持。`
8. `基于建构主义理论，设计让学生主动建构知识的活动。`

**提示词展示**:
- **基础推荐**：显示基础提示词和结构相关提示词（轮播显示3个）
- **学习科学**：单独展示学习科学提示词（显示4个，带绿色主题色）
- 用户点击"换一批"可以切换显示的提示词

### 5. 用户提问阶段

用户可以通过两种方式提问：

**方式1：使用预设提示词**
- 点击任意一个推荐的提示词按钮
- 提示词自动填充到输入框

**方式2：自定义输入**
- 在文本框中输入自己的问题
- 最少需要4个字符才能提交

### 7. AI 处理阶段

**AI 服务** (`backend/app/services/ai_qa.py`):

1. **构建提示词** (第107-200行)
   ```python
   def _build_prompt(self, question, context, lesson_title, cell_content):
       prompt_parts = []
       
       # 系统角色设定（增强学习科学理论支持）
       system_prompt = """你是一个专业的教学助手，擅长基于学习科学理论回答教育相关的问题。
       
       你熟悉以下学习科学理论和教学方法：
       
       1. **布鲁姆分类法（Bloom's Taxonomy）**：
          - 记忆、理解、应用、分析、评价、创造
       
       2. **苏格拉底式提问法（Socratic Questioning）**：
          - 澄清性问题、探索假设、证据和理由等
       
       3. **费曼学习法（Feynman Technique）**：
          - 选择概念并尝试解释、识别知识缺口等
       
       4. **5E教学模型**：
          - Engage、Explore、Explain、Elaborate、Evaluate
       
       5. **建构主义学习理论**：
          - 主动建构知识、最近发展区（ZPD）、脚手架支持
       
       6. **学习风格和差异化教学**：
          - 视觉型、听觉型、动觉型学习者、多元智能理论
       
       7. **元认知和主动输出**：
          - 自我监控、反思、通过输出促进深度学习"""
       prompt_parts.append(system_prompt)
       
       # 课程信息
       if lesson_title:
           prompt_parts.append(f"当前课程：{lesson_title}")
       
       # 上下文信息（包含教案结构）
       if context:
           prompt_parts.append(f"上下文信息：{context}")
       
       # 问题
       prompt_parts.append(f"教师问题：{question}")
       
       # 回答要求（包含学习科学理论应用指导）
       answer_guidance = """回答要求：
       1. 如果问题涉及教学设计，请结合学习科学理论提供建议
       2. 如果问题涉及提问技巧，请提供苏格拉底式提问的具体示例
       3. 如果问题涉及学生理解，请考虑费曼学习法和主动输出策略
       4. 如果问题涉及差异化教学，请考虑不同学习风格和多元智能
       5. 提供具体、可操作的建议，避免空泛的理论描述"""
       prompt_parts.append(answer_guidance)
       
       return "\n\n".join(prompt_parts)
   ```

2. **调用 AI API** (第150-196行)
   - 使用 OpenAI 兼容的 API（可配置）
   - 如果 API 不可用，会回退到模拟回答
   - 返回包含回答、置信度、响应时间等信息

3. **生成结构化反馈** (第109-179行)
   - 根据上下文生成洞察（Insights）
   - 生成建议行动（Actions）
   - 生成后续问题（Follow-up Questions）

### 1. 上下文感知
- AI 能够理解当前教案的结构和内容
- 基于教案标题和单元结构生成针对性建议

- 支持文本、代码、视频、活动等多种教案单元类型

## 学习科学理论详细说明

### 1. 布鲁姆分类法（Bloom's Taxonomy）

**理论概述**：
布鲁姆分类法将认知过程分为六个层次，从低阶思维到高阶思维：
- **记忆（Remember）**：回忆事实和基本概念
- **理解（Understand）**：解释概念和意义
- **应用（Apply）**：在新情境中使用信息
- **分析（Analyze）**：区分和组织信息
- **评价（Evaluate）**：做出判断和批判
- **创造（Create）**：产生新的或原创性的作品

**在 AI 助理中的应用**：
- 当教师选择"基于布鲁姆分类法设计认知层次活动"时，AI 会：
  - 分析当前教案内容
  - 为每个认知层次设计相应的活动
  - 确保学生能够从低阶思维逐步提升到高阶思维
  - 提供具体的活动示例和评估方法

**示例输出**：
```
基于布鲁姆分类法的活动设计：

1. 记忆层次：让学生列出智能农业的关键概念
2. 理解层次：要求学生解释传感器的工作原理
3. 应用层次：设计一个实际场景，让学生应用传感器知识
4. 分析层次：比较不同传感器类型的优缺点
5. 评价层次：评估某个智能农业方案的可行性
6. 创造层次：设计一个创新的智能农业系统
```

### 2. 苏格拉底式提问法（Socratic Questioning）

**理论概述**：
苏格拉底式提问通过一系列引导性问题，帮助学生逐步深入思考，而不是直接给出答案。主要包括：
- **澄清性问题**：帮助学生明确概念（"你所说的...是什么意思？"）
- **探索假设**：挑战学生的前提假设（"你假设了什么？"）
- **证据和理由**：要求学生提供支持证据（"有什么证据支持这个观点？"）
- **观点和视角**：考虑不同观点（"其他人可能怎么看这个问题？"）
- **影响和后果**：探索含义和后果（"如果这是真的，会有什么后果？"）
- **关于问题的问题**：元认知反思（"为什么这个问题很重要？"）

**在 AI 助理中的应用**：
- 当教师选择"设计苏格拉底式提问序列"时，AI 会：
  - 根据教案内容设计一系列引导性问题
  - 按照从简单到复杂的顺序组织问题
  - 为每个问题提供使用场景和预期效果
  - 帮助教师培养学生的批判性思维

**示例输出**：
```
苏格拉底式提问序列设计：

1. 澄清性问题：
   - "你所说的'智能农业'具体指什么？"
   - "传感器和传统农业工具有什么区别？"

2. 探索假设：
   - "你假设所有传感器都适合农业应用吗？"
   - "为什么你认为自动化一定更好？"

3. 证据和理由：
   - "有什么数据支持智能农业能提高产量？"
   - "你如何验证传感器的准确性？"

4. 观点和视角：
   - "农民可能如何看待智能农业？"
   - "环保主义者会有什么担忧？"

5. 影响和后果：
   - "如果智能农业普及，对传统农民有什么影响？"
   - "技术故障会导致什么后果？"

6. 元认知反思：
   - "为什么理解智能农业很重要？"
   - "这个问题如何与我们的生活相关？"
```

### 3. 费曼学习法（Feynman Technique）

**理论概述**：
费曼学习法强调通过"教别人"来促进深度学习，包括四个步骤：
1. **选择概念**：选择一个要学习的概念
2. **尝试解释**：用简单的语言向别人解释
3. **识别缺口**：发现解释不清楚的地方
4. **简化类比**：回到原始材料，用类比简化

**在 AI 助理中的应用**：
- 当教师选择"结合费曼学习法设计活动"时，AI 会：
  - 设计让学生用自己的话解释概念的活动
  - 提供"教别人"的场景（如向同学解释、制作教学视频等）
  - 设计识别知识缺口的反思环节
  - 鼓励使用类比和简化表达

**示例输出**：
```
费曼学习法活动设计：

活动1：向同学解释
- 让学生两两配对，互相解释"传感器的工作原理"
- 要求用最简单的语言，避免专业术语
- 如果对方听不懂，说明自己理解不够深入

活动2：制作教学视频
- 让学生制作3分钟的教学视频，解释一个概念
- 要求使用类比和生活中的例子
- 通过制作过程发现知识缺口

活动3：知识地图
- 让学生画出概念之间的关系图
- 用箭头和关键词说明连接关系
- 识别哪些地方解释不清楚

反思环节：
- "哪些地方你解释不清楚？"
- "你用了什么类比来帮助理解？"
- "如果重新解释，你会怎么说？"
```

### 4. 5E教学模型

**理论概述**：
5E教学模型是建构主义学习理论的实践模型，包括五个阶段：
- **Engage（参与）**：激发兴趣和动机，建立问题意识
- **Explore（探索）**：主动探索和发现，收集数据
- **Explain（解释）**：建构概念和解释，形成系统理解
- **Elaborate（深化）**：应用和迁移知识，解决新问题
- **Evaluate（评价）**：评估理解和反思，发展元认知

**在 AI 助理中的应用**：
- 当教师选择"基于5E教学模型优化"时，AI 会：
  - 分析当前教案的5E阶段分布
  - 为每个阶段提供优化建议
  - 设计缺失阶段的补充活动
  - 确保5E循环的完整性和连贯性

**示例输出**：
```
5E教学模型优化建议：

Engage（参与）环节：
- 当前：缺少引人入胜的引入
- 建议：添加一个认知冲突问题或有趣的现象
- 活动：展示智能农业的惊人数据，引发"为什么？"

Explore（探索）环节：
- 当前：探索活动不够深入
- 建议：增加动手操作和实验环节
- 活动：让学生操作PhET仿真，收集传感器数据

Explain（解释）环节：
- 当前：解释环节过于简单
- 建议：增加概念建构和讨论环节
- 活动：小组讨论后，用概念图展示理解

Elaborate（深化）环节：
- 当前：缺少应用和迁移
- 建议：设计新情境的应用任务
- 活动：设计一个智能农业方案，应用到新场景

Evaluate（评价）环节：
- 当前：评价方式单一
- 建议：增加自我评估和反思
- 活动：完成学习反思问卷，识别知识缺口
```

### 5. 建构主义学习理论

**理论概述**：
建构主义认为学习是学生主动建构知识的过程，强调：
- **主动建构**：学生不是被动接受，而是主动建构知识
- **基于经验**：新知识建立在已有经验基础上
- **社会互动**：通过与他人交流促进学习
- **最近发展区（ZPD）**：在他人帮助下能达到的水平
- **脚手架支持**：提供适当的支持和引导

**在 AI 助理中的应用**：
- 当教师选择"基于建构主义理论设计活动"时，AI 会：
  - 设计让学生主动探索和建构的活动
  - 考虑学生的前概念和经验
  - 设计协作学习和社会互动环节
  - 提供脚手架支持（提示、示例、引导性问题）

**示例输出**：
```
建构主义活动设计：

1. 激活前概念：
   - "关于传感器，你已经知道什么？"
   - 让学生画出自己的理解图

2. 主动探索：
   - 提供PhET仿真，让学生自由探索
   - 不直接告诉答案，让学生自己发现

3. 社会建构：
   - 小组讨论："你们发现了什么规律？"
   - 同伴互评："你觉得这个解释合理吗？"

4. 脚手架支持：
   - 提供提示卡："如果卡住了，看看提示"
   - 分层任务：基础任务 → 挑战任务
   - 引导性问题："这个现象让你想到什么？"

5. 概念建构：
   - 让学生用自己的话总结概念
   - 建立概念之间的联系
   - 修正错误的前概念
```

### 6. 学习风格和差异化教学

**理论概述**：
不同学生有不同的学习风格和智能类型：
- **学习风格**：视觉型、听觉型、动觉型
- **多元智能**：语言、逻辑数学、空间、音乐、身体运动、人际、内省、自然观察

**在 AI 助理中的应用**：
- 当教师选择"分析学习风格特点"时，AI 会：
  - 分析不同学习风格的特点
  - 为每种学习风格设计相应的活动
  - 提供差异化教学建议
  - 设计多元智能活动

**示例输出**：
```
差异化教学建议：

视觉型学习者：

听觉型学习者：

动觉型学习者：

多元智能活动：
```

### 7. 元认知和主动输出

**理论概述**：
元认知是对自己认知过程的认知，主动输出通过"输出倒逼输入"促进深度学习：
- **自我监控**：监控自己的学习过程
- **自我反思**：反思学习效果和策略
- **主动输出**：通过解释、写作、教学等方式输出知识
- **知识整合**：将新知识与已有知识整合

**在 AI 助理中的应用**：
- 当教师选择"设计主动输出活动"时，AI 会：
  - 设计让学生主动输出的活动（解释、写作、教学等）
  - 提供元认知反思环节
  - 设计自我监控和评估工具
  - 促进知识整合和迁移

**示例输出**：
```
主动输出活动设计：

1. 解释输出：
   - "用自己的话解释传感器的工作原理"
   - "向一个5岁孩子解释什么是传感器"

2. 写作输出：
   - 撰写学习笔记和总结
   - 写一篇关于传感器应用的文章
   - 制作学习日志

3. 教学输出：
   - 制作教学视频
   - 向同学讲解一个概念
   - 设计一个教学活动

4. 元认知反思：
   - "我是如何理解这个概念的？"
   - "哪些地方我还不太清楚？"
   - "我用了什么学习策略？"
   - "如果重新学习，我会怎么做？"

5. 自我监控：
   - 完成学习进度检查表
   - 评估自己的理解程度
   - 识别需要进一步学习的部分
```
