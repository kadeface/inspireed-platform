## 🚀 立即开始使用

### 方法 1：直接启动（最简单）

```bash
# 启动服务，会自动显示局域网访问地址
./start.sh

# 启动后会显示：
# - 本机访问地址（localhost）
# - 局域网访问地址（自动检测的 IP）
# - 移动设备访问提示
```

### 方法 2：自动配置（需要配置 CORS）

```bash
# 1. 运行配置脚本
./get-network-info.sh

# 2. 按提示选择自动配置（输入 y）

# 3. 重启服务
./restart.sh
```

### 方法 3：手动配置

```bash
# 1. 获取你的 IP 地址
ifconfig | grep "inet " | grep -v 127.0.0.1
# 假设得到: 192.168.1.100

# 2. 编辑后端配置
cd backend
cp env.example .env  # 如果还没有 .env 文件
# 编辑 .env，添加：
# BACKEND_CORS_ORIGINS=http://localhost:5173,http://192.168.1.100:5173

# 3. 编辑前端配置
cd ../frontend
cp env.example .env.local  # 如果还没有 .env.local 文件
# 编辑 .env.local，修改：
# VITE_API_BASE_URL=http://192.168.1.100:8000/api/v1

# 4. 启动服务
cd ..
./start.sh
```

## 🌐 访问地址

假设你的服务器 IP 是 `192.168.1.100`：

### 在服务器电脑上访问：
- 前端：http://localhost:5173
- 后端：http://localhost:8000
- API 文档：http://localhost:8000/docs

### 在局域网其他电脑上访问：
- 前端：http://192.168.1.100:5173
- 后端：http://192.168.1.100:8000
- API 文档：http://192.168.1.100:8000/docs

### 在手机/平板上访问：
连接同一 WiFi，然后访问：http://192.168.1.100:5173

## 🧪 快速测试

```
http://你的IP地址:8000/health
```

## ⚠️ 注意事项

1. **防火墙**：确保防火墙允许 5173 和 8000 端口
2. **同一网络**：所有设备必须连接到同一局域网
3. **IP 地址变化**：如果服务器 IP 改变，需要重新配置
4. **仅开发环境**：这个配置仅适用于开发和测试环境

## 🎯 常见问题

### Q: 如何找到我的 IP 地址？

**macOS/Linux:**
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

**Windows:**
```cmd
ipconfig
```

查找类似 `192.168.x.x` 的地址。

### Q: 其他电脑访问不了怎么办？

1. 确认服务正在运行：`curl http://localhost:8000/health`
2. 确认防火墙设置
3. 使用 ping 测试网络连通性：`ping 你的IP地址`
4. 检查配置文件是否正确
5. 查看日志：`tail -f logs/backend.log`

### Q: 出现 CORS 错误怎么办？

确保 `backend/.env` 中包含：
```
BACKEND_CORS_ORIGINS=http://你的IP:5173
```

然后重启服务：`./restart.sh`

### Q: 可以在手机上访问吗？

可以！确保手机连接到同一 WiFi，然后访问 `http://你的IP:5173`
