# 网络连接问题解决方案

## 问题现象

在使用系统时，可能会遇到以下错误：
```
ERR_CONNECTION_TIMED_OUT
[vite] server connection lost. Polling for restart...
```

## 问题原因

1. **Mac 电脑进入睡眠**：系统自动进入睡眠模式导致网络服务中断
2. **WiFi 连接中断**：无线网络暂时断开
3. **网络不稳定**：局域网连接质量问题

## 解决方案

### 方案1：刷新页面（最简单）

只需刷新浏览器页面即可恢复连接：
- **Windows/Linux**：按 `F5` 或 `Ctrl+R`
- **macOS**：按 `Cmd+R`

### 方案2：防止 Mac 进入睡眠

如果您的 Mac 经常自动睡眠，可以运行防睡眠脚本：

```bash
cd /Users/382241106qq.com/inspireed-platform-main
./prevent-sleep.sh
```

**注意**：
- 运行此脚本后，您的 Mac 将保持唤醒状态
- 如需停止，按 `Ctrl+C`
- 或者直接关闭终端窗口

### 方案3：调整系统睡眠设置

**临时设置（推荐）**：
```bash
# 防止系统睡眠（当前会话有效）
sudo pmset -a disablesleep 1

# 恢复正常睡眠设置
sudo pmset -a disablesleep 0
```

**永久设置（通过系统偏好设置）**：
1. 打开 **系统偏好设置**
2. 选择 **节能**（或 **电池**）
3. 将"电脑睡眠"滑块调整为较长时间（如 1 小时或"永不"）
4. 取消勾选"显示器关闭时，防止自动进入睡眠"

### 方案4：检查网络连接

确保：
- ✅ 所有设备连接到同一个 WiFi 网络
- ✅ WiFi 信号强度良好
- ✅ 路由器工作正常

### 方案5：重启服务

如果问题持续，可以重启所有服务：

```bash
cd /Users/382241106qq.com/inspireed-platform-main
./restart.sh
```

## 预防措施

1. **使用有线网络**：如果条件允许，使用以太网连接更稳定
2. **保持电脑唤醒**：在使用系统时，运行 `prevent-sleep.sh` 脚本
3. **定期保存**：虽然系统有自动保存功能，但重要修改后建议手动保存

## 技术说明

**ERR_CONNECTION_TIMED_OUT** 错误表示：
- 前端无法连接到后端服务器（`http://192.168.1.103:8000`）
- 可能是网络中断、服务器睡眠或防火墙阻止

**[vite] server connection lost** 表示：
- 前端开发服务器与浏览器的 WebSocket 连接断开
- 通常是因为系统睡眠或网络中断导致
