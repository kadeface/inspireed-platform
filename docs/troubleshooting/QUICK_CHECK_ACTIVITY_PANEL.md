# 活动提交面板快速检查清单

## 🔍 当前问题
在教师端的课堂模式下，看不到学生提交的答案面板。

## ✅ 检查清单

按照以下步骤逐一检查：

### □ 1. 已开始课堂会话
- [ ] 点击了"开始上课"按钮
- [ ] 状态栏显示"上课中"（绿色）

### □ 2. 已选择活动模块
- [ ] 在导播台中点击了活动类型的 Cell（紫色图标 📋）
- [ ] 该模块显示蓝色背景（已选中）

### □ 3. 已启动活动
- [ ] 看到了"🎯 开始活动"按钮
- [ ] 点击了"开始活动"按钮
- [ ] 按钮变成了"✅ 结束活动"
- [ ] 活动图标上出现了 🎯 标记

### □ 4. 教案已保存
- [ ] 教案已保存到数据库（不是草稿）
- [ ] 活动 Cell 有数字 ID（不是 UUID）

### □ 5. 查看浏览器控制台
打开浏览器开发者工具（按 F12），查看控制台：

- [ ] 没有红色错误信息
- [ ] 看到 "✅ 活动启动成功" 消息
- [ ] 看到 "✅ 找到活动 Cell" 消息

---

## 🎯 正确的操作流程

**步骤 1：开始上课**
```
点击 "▶️ 开始上课" → 状态变为 "上课中"
```

**步骤 2：选择活动**
```
导播台 → 点击活动模块 → 模块变蓝色
```

**步骤 3：启动活动**
```
当前模块详情 → 点击 "🎯 开始活动" → 按钮变为 "✅ 结束活动"
```

**步骤 4：查看提交**
```
面板自动出现 → 显示 "📝 活动提交" → 列表显示学生提交
```

## 🐛 调试模式

### 打开浏览器控制台（F12）

您现在应该能看到详细的调试信息：

#### ✅ 成功的日志输出示例：
```
🎬 ClassroomControlBoard.handleStartActivity 被调用
✅ 发送 startActivity 事件, cellId: 123
🎯 handleStartActivity 被调用
📤 准备启动活动，Cell ID: 123
✅ 活动启动成功
📊 活动提交面板应该显示了
🔍 计算 currentActivityCell
✅ 找到活动 Cell
📝 加载活动提交: 3 份
```

#### ❌ 失败的日志输出示例：

**情况 1: Cell ID 无效**
```
❌ Cell ID 无效，无法启动活动
```
👉 **解决**: 保存教案后刷新页面

**情况 2: 未找到活动 Cell**
```
❌ 未找到匹配的活动 Cell
```
👉 **解决**: 检查教案内容，重新保存

**情况 3: 没有选中 Cell**
```
❌ 无法确定要启动的活动 Cell ID
```
👉 **解决**: 先在导播台点击活动模块

## ❓ 仍然无法解决？

### 方法 1: 导出调试信息

在浏览器控制台运行：
```javascript
console.log('Session:', window.$session)
console.log('Current Activity ID:', window.$session?.current_activity_id)
console.log('Lesson Content:', window.$lesson?.content)
```

### 方法 2: 截图以下内容

1. 整个教师控制面板界面
2. 浏览器控制台的所有日志
3. Network 标签中的请求列表（特别是带 `activity` 的请求）

### 方法 3: 重启流程

1. 结束课程
2. 退出登录
3. 清除浏览器缓存
4. 重新登录
5. 开始上课
6. 重新尝试
