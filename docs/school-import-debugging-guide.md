# 学校批量导入功能调试指南

## 400错误排查步骤

### 1. 检查浏览器控制台

打开浏览器开发者工具（F12），查看：
- **Network标签**：找到失败的请求
- **Request Payload**：检查FormData是否正确
- **Response**：查看后端返回的具体错误信息

### 2. 检查后端日志

查看后端终端输出，查找：
- `导入学校失败`
- `Excel解析失败`
- `文件上传失败`

### 3. 常见400错误原因

#### 错误1：文件格式不正确

**错误信息**：`只支持Excel文件格式 (.xlsx, .xls)`

**原因**：
- 上传了CSV文件（虽然模板是CSV，但导入需要Excel格式）
- 文件扩展名不正确

**解决方案**：
1. 下载模板后，用Excel打开
2. 填写数据
3. 另存为 `.xlsx` 格式
4. 重新上传

---

#### 错误2：文件内容为空

**错误信息**：`上传的文件为空` 或 `Excel文件为空或没有工作表`

**原因**：
- 文件损坏
- 文件确实是空的

**解决方案**：
1. 重新下载模板
2. 确保文件有内容
3. 检查文件大小（应该 > 0）

---

#### 错误3：缺少必需列

**错误信息**：`缺少必需列: ['region_name', 'school_name']`

**原因**：
- Excel表头不正确
- 列名不匹配

**解决方案**：
1. 检查表头第一行
2. 确保包含：`区域名称` 和 `学校名称`
3. 列名必须完全匹配（区分大小写）

**支持的列名**：
- 区域名称：`区域名称`、`市(区)`、`区域`、`市`、`区`
- 学校名称：`学校名称`、`学校`
- 学校代码：`学校代码`、`代码`
- 学校类型：`学校类型`、`类型`
- 地址：`地址`、`学校地址`
- 联系电话：`联系电话`、`电话`、`联系方式`
- 邮箱：`邮箱`、`Email`、`email`
- 校长：`校长`、`校长姓名`

---

#### 错误4：数据验证失败

**错误信息**：`区域名称不能为空` 或 `学校名称不能为空`

**原因**：
- 数据行中必填字段为空
- 数据格式不正确

**解决方案**：
1. 检查Excel数据
2. 确保所有行的"区域名称"和"学校名称"都有值
3. 删除空行

---

#### 错误5：文件读取失败

**错误信息**：`解析Excel文件失败: ...`

**原因**：
- 文件损坏
- 文件格式问题
- openpyxl库问题

**解决方案**：
1. 用Excel重新保存文件
2. 确保文件格式为 `.xlsx` 或 `.xls`
3. 检查文件是否损坏

---

### 4. 测试步骤

#### 步骤1：验证文件格式

```bash
# 检查文件类型
file 学校信息导入模板.xlsx
# 应该显示：Microsoft Excel 2007+

# 检查文件大小
ls -lh 学校信息导入模板.xlsx
# 应该 > 0 字节
```

#### 步骤2：验证Excel内容

用Excel打开文件，检查：
- 第一行是表头
- 表头包含"区域名称"和"学校名称"
- 至少有一行数据
- 数据行的必填字段都有值

#### 步骤3：测试最小数据

创建一个最简单的测试文件：
```
区域名称,学校名称
北京市,测试学校1
```

保存为 `.xlsx` 格式，尝试导入。

---

### 5. 调试技巧

#### 查看请求详情

在浏览器Network标签中：
1. 找到失败的请求
2. 点击查看详情
3. 检查：
   - Request Headers（Content-Type应该是 `multipart/form-data`）
   - Request Payload（应该包含file字段）
   - Response（查看具体错误信息）

#### 后端调试

在后端代码中添加日志：

```python
# 在 admin_organization.py 的 import_schools 函数中
logger.info(f"收到文件上传请求: filename={file.filename}, size={file.size}")
logger.info(f"文件扩展名: {file_ext}")
logger.info(f"auto_create_region: {auto_create_region}")
```

#### 测试API端点

使用curl测试：

```bash
curl -X POST "http://localhost:8000/api/v1/admin/organization/schools/import?auto_create_region=true" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@学校信息导入模板.xlsx"
```

---

### 6. 快速修复检查清单

- [ ] 文件格式是 `.xlsx` 或 `.xls`（不是 `.csv`）
- [ ] 文件大小 > 0
- [ ] 表头第一行包含"区域名称"和"学校名称"
- [ ] 至少有一行数据
- [ ] 数据行的"区域名称"和"学校名称"都有值
- [ ] 浏览器控制台没有其他错误
- [ ] 后端服务正在运行
- [ ] 已登录管理员账号

---

### 7. 如果仍然失败

1. **查看完整错误信息**：
   - 浏览器控制台的Network标签
   - 后端终端输出

2. **检查文件内容**：
   - 用Excel打开文件
   - 检查是否有隐藏字符
   - 检查编码问题

3. **尝试最小测试**：
   - 只包含一行数据
   - 只填必填字段
   - 使用简单的区域名称（如"北京市"）

4. **检查后端依赖**：
   ```bash
   cd backend
   pip list | grep openpyxl
   # 应该显示 openpyxl 已安装
   ```

## 成功导入的标志

导入成功后，应该看到：
- ✅ 导入结果对话框显示统计信息
- ✅ 成功数 > 0
- ✅ 学校列表中可以看到新导入的学校
- ✅ 如果创建了区域，区域列表中可以看到新区域
