# 学校批量导入功能测试指南

## 测试环境准备

### 1. 启动服务

```bash
# 启动后端
cd backend
source venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 启动前端
cd frontend
pnpm dev
```

### 2. 登录管理员账号

- 访问：http://localhost:5173
- 使用管理员账号登录（如：admin@inspireed.com / admin123）

### 3. 进入组织架构管理页面

- 导航到：管理员 → 组织架构管理
- 切换到"学校管理"标签页

## 测试场景

### 场景1：下载模板

**步骤**：
1. 点击"批量导入"按钮
2. 在导入对话框中，点击"下载Excel模板"按钮
3. 检查下载的文件

**预期结果**：
- ✅ 成功下载 `学校信息导入模板.csv` 文件
- ✅ 文件包含表头：区域名称*、学校名称*、学校代码、学校类型、地址、联系电话、邮箱、校长
- ✅ 包含示例数据（2行）

**验证点**：
- 文件格式正确（CSV，UTF-8 BOM编码）
- 字段顺序正确
- 示例数据合理

---

### 场景2：导入新学校（区域已存在）

**准备数据**：
1. 确保系统中已存在区域（如：北京市）
2. 准备Excel文件，包含：
   - 区域名称：北京市（已存在）
   - 学校名称：北京市第一中学（新学校）
   - 学校代码：10001
   - 学校类型：高中
   - 其他可选字段

**步骤**：
1. 点击"批量导入"按钮
2. 跳过步骤1（或下载模板后填写）
3. 在步骤2中：
   - 选择"自动创建区域"：关闭（因为区域已存在）
   - 上传Excel文件
4. 点击"开始导入"

**预期结果**：
- ✅ 导入成功
- ✅ 显示统计：成功1条，创建学校1所
- ✅ 学校列表中可以看到新导入的学校
- ✅ 学校信息完整（名称、代码、类型等）

**验证点**：
- 学校创建成功
- 区域关联正确
- 可选字段保存正确

## 常见问题排查

### 问题1：导入失败，提示"解析Excel文件失败"

**可能原因**：
- 文件格式不正确
- 文件损坏
- 文件编码问题

**解决方案**：
- 确保使用 `.xlsx` 或 `.xls` 格式
- 重新下载模板，使用模板填写
- 检查文件是否损坏

---

### 问题2：导入成功但学校未创建

**可能原因**：
- 学校已存在，被跳过
- 区域不存在且未开启自动创建
- 数据验证失败

**解决方案**：
- 查看导入结果统计
- 检查错误详情表格
- 确认区域是否存在

---

### 问题3：区域自动创建失败

**可能原因**：
- 区域代码生成冲突
- 数据库约束错误

**解决方案**：
- 查看后端日志
- 检查区域表数据
- 手动创建区域后重试

---

### 问题4：前端显示错误但后端成功

**可能原因**：
- API响应格式不匹配
- 前端错误处理问题

**解决方案**：
- 检查浏览器控制台
- 查看网络请求响应
- 检查API返回格式
