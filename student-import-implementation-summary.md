## 🎯 问题诊断

### 原始问题
用户报告："设置科目的下一步，考场安排，下一步没法点击"

### 根本原因
```
考场安排步骤 → 点击"自动编排考场" → 400错误
                                    ↓
                              找不到学生数据
                                    ↓
                    数据库中没有学生账户（User表为空）
```

**为什么没有学生？**
- 原有的"学生导入"功能只是创建**考号映射**（ExamNumberMapping）
- 不是创建**学生账户**（User表）
- 学生账户必须先存在，才能创建考号映射

---

## ✅ 解决方案

### 创建学生账户批量导入功能

**新导入类型**: `student_account`

**核心特性**:
1. ✅ 使用友好格式（学校名称、年级级别、班级编号）
2. ✅ 双模式支持（区县管理员/学校管理员）
3. ✅ 自动生成用户名、邮箱
4. ✅ 自动关联班级、年级、学校
5. ✅ 默认密码：`123456`
6. ✅ 支持更新已存在的学生

## 📊 功能对比

### 旧功能：学生考号导入（student）

**用途**: 将已有学生与考号关联
**前提**: 学生账户必须已存在
**字段**: 学校名称、班级编号、学籍号、考号、姓名
**问题**: 无法创建新学生账户

### 新功能：学生账户导入（student_account）✨

**用途**: 批量创建学生账户
**前提**: 学校、年级、班级已存在
**字段**: 学校名称、年级级别、班级编号、学籍号、姓名
**优势**: 可直接创建学生账户

## 🎯 完整使用流程

### 步骤1：准备组织架构

```bash
# 导入学校
POST /api/v1/import?strategy_type=school
file: schools.xlsx

# 导入班级
POST /api/v1/import?strategy_type=classroom
file: classrooms.xlsx
```

### 步骤2：导入学生账户

```bash
POST /api/v1/import?strategy_type=student_account
file: students.xlsx
```

### 步骤3：创建考试

```bash
POST /api/v1/exams
{
  "name": "2024年春季期末考试",
  "grade_id": 3,
  "exam_date": "2024-06-20 12:00:00"
}
```

### 步骤4：分配考场 ✅

```bash
POST /api/v1/exams/{exam_id}/rooms/auto-assign
{
  "capacity_per_room": 30,
  "arrangement_type": "by_class"
}
```

**结果**: 成功创建考场，学生已存在！

## 🔍 错误处理

### 常见错误及解决方案

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| 学校 'XXX' 未找到 | 学校不存在 | 先用school导入创建学校 |
| 年级级别 X 不存在 | 年级未创建 | 在系统中创建对应年级 |
| 班级编号 'XXX' 不存在 | 班级未创建 | 先用classroom导入创建班级 |
| 学籍号 'XXX' 已存在 | 学生已存在 | 设置`update_existing=true` |
