# InspireEd Platform - 用户、班级、学校关联操作指南

## 📊 系统用户关系模型

### 1️⃣ 用户表（User）的直接关联字段

| 字段 | 说明 | 适用对象 | 关系类型 |
|------|------|---------|---------|
| `school_id` | 所属学校 | 教师、学生 | 多对一（一个用户属于一个学校） |
| `classroom_id` | 主班级/默认班级 | 主要是学生 | 多对一（一个学生属于一个班级） |

### 2️⃣ 班级成员表（ClassroomMembership）的多对多关联

| 字段 | 说明 | 适用对象 | 关系类型 |
|------|------|---------|---------|
| `classroom_id` | 班级ID | 教师、学生 | 多对多 |
| `user_id` | 用户ID | 教师、学生 | 多对多 |
| `role_in_class` | 班级角色 | 教师、学生 | 枚举值 |
| `is_active` | 是否激活 | 教师、学生 | 布尔值 |
| `is_primary_class` | 是否为主班级 | 主要是学生 | 布尔值 |

### 3️⃣ 班级角色类型（role_in_class）

| 角色代码 | 角色名称 | 适用对象 | 权限说明 |
|---------|---------|---------|---------|
| `head_teacher_primary` | 正班主任 | 教师 | 班级管理权限，可添加成员 |
| `head_teacher_deputy` | 副班主任 | 教师 | 班级管理权限，可添加成员 |
| `subject_teacher` | 任课教师 | 教师 | 可在该班级上课、发布教案 |
| `student` | 学生 | 学生 | 班级学生 |
| `cadre` | 班干部 | 学生 | 班级干部学生 |

## 🎯 关联设置场景和对应页面

### 场景1：将学生分配到班级（主班级）

**用途**：设置学生的主班级（默认班级），学生登录后自动进入该班级

**设置页面**：`/admin/user-management` → **用户管理**

**操作步骤**：
1. 进入"用户管理"页面
2. 找到目标学生账号
3. 点击"编辑"按钮
4. 填写以下字段：
   - **所属学校**（必填）：选择学校
   - **年级**（可选）：选择年级
   - **班级**（必填）：选择班级
5. 点击"保存"

**效果**：
- 用户的 `User.classroom_id` 字段被设置
- 学生登录后自动关联到该班级

**适用情况**：
- ✅ 新生入学分配班级
- ✅ 学生转班（主班级变更）
- ✅ 快速批量分配（导入用户时）

---

### 场景2：将教师分配到学校

**用途**：设置教师的所属学校，教师只能看到自己学校的班级

**设置页面**：`/admin/user-management` → **用户管理**

**操作步骤**：
1. 进入"用户管理"页面
2. 找到目标教师账号
3. 点击"编辑"按钮
4. 选择**所属学校**
5. 点击"保存"

**效果**：
- 用户的 `User.school_id` 字段被设置
- 教师在"可用班级列表"中只能看到该学校的班级

**适用情况**：
- ✅ 新教师入职
- ✅ 教师调动到新学校

---

### 场景3：添加教师到班级（多班级教学）⭐推荐

**用途**：让教师可以在指定班级上课、发布教案（支持一个教师教多个班级）

**设置页面**：`/admin/organization-management` → **班级管理** → **成员管理**

**操作步骤**：
1. 进入"组织管理" → "班级管理"
2. 找到目标班级（如"七年级一班"）
3. 点击"成员管理"按钮
4. 点击"+ 添加成员"
5. 搜索教师（输入用户名或姓名）
6. 选择教师后，设置：
   - **角色**：选择 `subject_teacher`（任课教师）或其他角色
   - **是否主班级**：不适用（仅对学生有效）
7. 点击"保存"

**效果**：
- 在 `ClassroomMembership` 表中创建记录
- 教师可以在该班级创建课堂会话
- 教师可以发布教案到该班级

**适用情况**：
- ✅ 一个教师教多个班级（如数学老师教3个班）
- ✅ 代课教师临时添加
- ✅ 班主任设置

**API调用示例**：
```bash
POST /api/v1/classroom-assistant/classrooms/{classroom_id}/members
{
  "user_id": 39,  # 教师ID
  "role_in_class": "subject_teacher",  # 任课教师
  "is_active": true
}
```

### 场景4：将学生添加到多个班级

**用途**：支持学生同时在多个班级（如选修课、兴趣班）

**设置页面**：`/admin/organization-management` → **班级管理** → **成员管理**

**操作步骤**：同场景3，但角色选择 `student`

**注意事项**：
- 可以设置其中一个班级为"主班级"（`is_primary_class = true`）
- 学生的主班级应该在 `User.classroom_id` 中设置
- 其他班级通过 `ClassroomMembership` 添加

---

### 场景5：批量添加学生到班级

**用途**：快速将一个班级的所有学生复制到另一个班级

**设置页面**：`/admin/organization-management` → **班级管理** → **成员管理**

**操作步骤**：
1. 进入目标班级的"成员管理"
2. 点击"批量添加"
3. 选择"从其他班级复制"
4. 选择源班级
5. 勾选要复制的所有学生
6. 点击"批量添加"

## ⚠️ 重要提示

### 1. 两种关联方式的区别

| 方式 | User表字段 | ClassroomMembership表 | 优先级 |
|------|-----------|---------------------|-------|
| **主班级** | `User.classroom_id` | - + `is_primary_class=true` | 高 |
| **多班级** | - | `ClassroomMembership` | 低 |

**推荐做法**：
- 学生主班级：设置 `User.classroom_id`
- 多班级支持：通过 `ClassroomMembership` 添加
- 教师任教班级：统一通过 `ClassroomMembership` 添加

### 2. 教师上课权限的检查逻辑

教师能否在某个班级上课，取决于：
1. ✅ 教师的 `User.school_id` 必须匹配班级的 `Classroom.school_id` **或者**
2. ✅ 教师在 `ClassroomMembership` 中有该班级的记录（role_in_class 为教师角色）

### 3. 学生加入课堂的检查逻辑

学生能否加入某个班级的课堂，取决于：
1. ✅ 学生的 `User.classroom_id` 匹配课堂的班级ID **或者**
2. ✅ 学生的 `User.school_id` 匹配班级的 `Classroom.school_id` 且该班级支持多学生加入

## 🚀 快速操作

### 场景：新学期快速分配

**批量导入学生**：
1. 准备Excel文件（用户名、姓名、学校、班级）
2. 使用"用户管理" → "批量导入"
3. 系统自动设置 `User.classroom_id`

**批量分配教师**：
1. 进入"班级管理"
2. 对每个班级点击"成员管理"
3. 使用"批量添加"功能
4. 或使用脚本批量执行SQL

## 📞 常见问题

### Q1: 教师看不到某个班级？
**A**: 检查：
1. 教师的 `school_id` 是否与班级的 `school_id` 匹配
2. 或者教师在 `ClassroomMembership` 中是否有该班级的记录

### Q2: 学生加入课堂失败？
**A**: 检查：
1. 学生的 `classroom_id` 是否正确
2. 或者学生的 `school_id` 是否与课堂班级的学校匹配

### Q3: 如何移除教师的班级关联？
**A**:
1. 进入"班级管理" → 目标班级 → "成员管理"
2. 找到该教师
3. 点击"移除"
4. 或者将 `is_active` 设为 `false`
