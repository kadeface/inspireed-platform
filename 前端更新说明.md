# 前端更新后需要刷新浏览器

## 问题
修改了前端代码，但浏览器中显示的还是旧版本。

## 解决方案

### 方法1：硬刷新浏览器（推荐）

**Windows/Linux**:
```
Ctrl + Shift + R
或
Ctrl + F5
```

**Mac**:
```
Cmd + Shift + R
```

### 方法2：清除浏览器缓存

1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 方法3：重启前端服务

如果硬刷新无效，重启前端：

```bash
# 停止前端（Ctrl+C）
# 然后重新启动
cd frontend
pnpm dev
```

### 方法4：检查浏览器控制台

1. 打开开发者工具（F12）
2. 查看 Console 标签
3. 查看是否有错误信息
4. 查看 Network 标签，确认文件是否重新加载

## Vite 热重载

正常情况下，Vite 会自动热重载（HMR），但有时候可能失败：
- 修改了 `<script setup>` 部分时
- 修改了响应式数据时
- 浏览器缓存问题时

此时需要手动刷新浏览器。

## 验证修改是否生效

刷新后，打开"创建教学任务"表单，应该看到：

**已移除**：
- ❌ "学年"输入框

**新增**：
- ✅ 学期选择器下方显示"学年：2024-2025"
- ✅ 表格移除了"学年"列（从10列变为9列）
- ✅ 教师选择器显示所属学校

如果还是看不到变化，请提供：
1. 浏览器控制台的错误信息
2. Network 标签中加载的文件时间戳
