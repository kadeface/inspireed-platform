## 关键组件

### 前端组件

1. **LessonView.vue**
   - 学生端教案页面
   - 页面加载时自动调用 `findAndJoinSession()`

2. **useClassroomSession.ts**
   - 课堂会话 Composable
   - 提供 `findAndJoinSession()` 函数
   - 管理会话状态和 WebSocket 连接

3. **classroomSession.ts**
   - API 服务层
   - 提供 `listSessions()` 和 `joinSession()` 方法

### 后端 API

1. **GET /lessons/{lesson_id}/sessions**
   - 查询教案的所有会话
   - 支持按状态过滤

2. **POST /sessions/{session_id}/join**
   - 学生加入会话
   - 包含完整的权限验证

3. **WebSocket /sessions/{session_id}/ws**
   - WebSocket 连接端点
   - 实时同步会话状态

### 数据模型

1. **ClassSession**
   - 课堂会话
   - 包含状态、当前Cell、学生统计等

2. **StudentSessionParticipation**
   - 学生参与记录
   - 包含加入时间、活跃状态、进度等

3. **ClassroomMembership**
   - 班级成员关系
   - 支持多班级和角色管理

### 工具函数

1. **check_user_in_classroom()**
   - 统一检查用户是否属于指定班级
   - 支持多班级和向后兼容

2. **get_user_classroom_ids()**
   - 获取用户所属的所有班级ID
   - 优先从 ClassroomMembership 获取

## 权限验证流程

### 后端权限检查（join_session）

1. **角色检查**
   ```python
   if current_role != UserRole.STUDENT:
       raise HTTPException(403, "只有学生可以加入会话")
   ```

2. **会话存在性检查**
   ```python
   session = await db.get(ClassSession, session_id)
   if not session:
       raise HTTPException(404, "会话不存在")
   ```

3. **会话状态检查**
   ```python
   if session.status == ClassSessionStatus.ENDED:
       raise HTTPException(400, "会话已结束")
   ```

4. **班级成员关系检查**
   ```python
   has_access = await check_user_in_classroom(db, current_user, classroom_id)
   if not has_access:
       raise HTTPException(403, "无权加入该会话")
   ```

### WebSocket 权限检查

1. Token 验证
2. 角色检查（必须是学生）
3. 会话存在性检查
4. 会话状态检查（不能是 ended）
5. 班级成员关系检查

## 错误处理

### 前端错误处理

- **403 权限错误**: 不重试，直接返回 null
- **400 会话已结束**: 不重试，直接返回 null
- **其他错误**: 最多重试 3 次，每次延迟 2 秒
- **WebSocket 失败**: 降级到轮询模式，不阻塞页面

### 后端错误响应

- **403**: 权限不足（角色错误、班级不匹配）
- **404**: 资源不存在（会话不存在）
- **400**: 状态错误（会话已结束）

## 数据一致性

### 并发加入场景

- 使用数据库唯一约束：`UniqueConstraint("session_id", "student_id")`
- 如果并发创建，数据库会抛出唯一约束错误
- 后端会捕获错误并更新现有记录

### 重新加入场景

- 如果学生已加入但 `is_active=False`，会更新为 `True`
- 更新 `last_active_at` 时间戳
- 如果之前是离线状态，增加 `active_students` 计数

## 降级方案

### WebSocket 失败处理

1. 连接失败时，自动降级到轮询模式
2. 轮询间隔：5 秒
3. 轮询内容：调用 `getSession()` API 获取会话状态更新
4. 不阻塞页面加载和显示

## 关键发现

1. **权限检查统一化**: 使用 `check_user_in_classroom()` 统一检查班级成员关系，支持多班级和向后兼容

2. **会话选择策略**: 优先选择 active 状态，其次 pending 状态，按 ID 降序排序选择最新会话

3. **重试机制**: 前端支持最多 3 次重试，特殊错误（403、400）不重试

4. **降级方案**: WebSocket 失败时自动降级到轮询模式，保证功能可用性

5. **数据一致性**: 使用数据库唯一约束防止重复加入，支持重新加入场景
