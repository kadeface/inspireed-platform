## 🔄 核心变化

### 1. 角色扩展：从三方到四方

**之前**：教师 → 学生 → 系统

**现在**：教研者 → 教师 → 学生 → 系统（闭环）

| 角色 | 职责 | 关键词 |
|-----|------|-------|
| **教研者** | 课程体系构建者 | 知识图谱、学习路径、课程规划 |
| **教师** | 活动单元创建者 | 互动组件、活动设计、内容创建 |
| **学生** | 探究式学习者 | 操作实验、完成任务、主动构建 |
| **系统** | 数据闭环驱动者 | 采集数据、分析洞察、闭环反馈 |

### 2. 术语精准化

| 之前 | 现在 | 原因 |
|-----|------|-----|
| 教案 | 活动单元 (Activity Unit) | 更强调结构化和互动性 |
| 学习活动手册 | 活动单元 + 活动手册 | 区分单个活动和完整手册 |
| 数据驱动 | 数据闭环反馈 | 强调完整的闭环过程 |
| - | 课程体系 | 新增顶层概念 |

### 3. 流程完整化

**之前的流程**（线性）：
```
设计 → 使用 → 数据分析
```

**现在的流程**（闭环）：
```
课程体系构建 
    ↓
活动单元创建
    ↓
探究式学习
    ↓
数据采集分析
    ↓
闭环反馈 → 优化课程体系/活动单元
```

## 💡 关键概念澄清

### 活动单元 vs 活动手册

**活动单元（Activity Unit）**
- 单个完整的学习活动
- 例如："牛顿第一定律探究实验"
- 包含多个Cell（文本、仿真、问答等）

**活动手册（Activity Workbook）**
- 由多个活动单元组成
- 例如："力学单元活动手册"
- 相当于一节课或一个章节的完整材料

### 数据闭环

**闭环四步骤**：
1. **采集** - 自动记录学习行为数据
2. **分析** - 统计分析和AI洞察
3. **反馈** - 向教师和教研者呈现洞察
4. **改进** - 基于数据优化课程和活动

**与传统"数据驱动"的区别**：
- 传统：数据 → 报表（单向）
- 闭环：数据 → 分析 → 反馈 → 改进 → 再采集（循环）
