# Alembic 迁移文件枚举类型问题分析

## 🔍 发现的问题

### 问题 1: PostgreSQL 不支持 `IF NOT EXISTS` 语法（严重错误）

**文件**: `008_add_activity_system.py` (第 21-22 行)

```python
# ❌ 错误：PostgreSQL 不支持这个语法
op.execute("ALTER TYPE celltype ADD VALUE IF NOT EXISTS 'activity'")
op.execute("ALTER TYPE celltype ADD VALUE IF NOT EXISTS 'flowchart'")
```

**错误原因**：
- PostgreSQL 的 `ALTER TYPE ... ADD VALUE` 命令**不支持** `IF NOT EXISTS` 子句
- 这会导致迁移失败，报错：`syntax error at or near "IF"`

**正确的做法**（参考其他迁移文件）：
```python
# ✅ 正确：使用 DO 块检查
op.execute("""
    DO $$ 
    BEGIN
        IF NOT EXISTS (
            SELECT 1 FROM pg_enum 
            WHERE enumlabel = 'activity' 
            AND enumtypid = (SELECT oid FROM pg_type WHERE typname = 'celltype')
        ) THEN
            ALTER TYPE celltype ADD VALUE 'activity';
        END IF;
    END $$;
""")
```

### 问题 3: create_type=False 可能导致错误

**涉及文件**：
- `005_add_qa_system.py` (第 75 行)
- `006_add_learning_science_fields.py` (第 34, 43 行)
- `008_add_activity_system.py` (第 64, 153 行)
- `009_add_subject_groups.py` (第 46, 114 行)
- `016_add_classroom_session_tables.py` (第 43 行)
- `004_add_student_enhancement_features.py` (第 88, 140 行)
- `20250122_add_student_projects.py` (第 52, 142, 155 行)

**问题**：
```python
postgresql.ENUM(..., name="celltype", create_type=False)
```

- `create_type=False` 表示不创建枚举类型，假设已存在
- 如果枚举类型不存在，会导致错误：`type "celltype" does not exist`

**解决方案**：
1. 在创建表之前，确保枚举类型已创建（使用 DO 块）
2. 或者使用 `create_type=True`（但要注意重复创建的问题）
