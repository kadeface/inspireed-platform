## 🎯 项目概述

**InspireEd** 是一个面向STEM教育的**探究式学习与教学平台**，基于建构主义学习理论、5E教学模型和循证教育理念，构建完整的PDCA教学质量管理循环。

### 一句话介绍
> 基于学习科学的智能教学平台，让STEM教育从"教知识"转向"培养能力"，从"结果评价"转向"过程优化"。

### 项目定位
- **目标用户**：K-12及高等教育阶段的STEM教师、学生、教研员
- **核心使命**：通过技术赋能，实现科学化、个性化的探究式教学
- **价值追求**：让每个学生都能在主动探索中建构知识，发展能力

## 🧠 理论框架

### 双循环协同体系

#### 🔄 PDCA教学质量管理循环（教师视角）

```
┌─────────────────────────────────────────────────┐
│         PDCA 教学质量管理循环                    │
├─────────────────────────────────────────────────┤
│                                                  │
│  📋 Plan (系统化设计)                            │
│     ↓                                           │
│  🚀 Do (结构化实施)                              │
│     ↓                                           │
│  ✅ Check (过程性评估)                           │
│     ↓                                           │
│  🔄 Act (循证改进)                               │
│     ↑                                           │
│     └───────────────────────────────────────────┘
│                                                  │
│  基于布鲁姆分类法、能力本位框架、学习科学理论    │
└─────────────────────────────────────────────────┘
```

**Plan（系统化设计）**
- 基于布鲁姆分类法构建层次化知识图谱
- 能力本位框架设计学习目标体系
- 5E教学模型规划活动序列

**Do（结构化实施）**
- 多模态教学资源整合
- 符合学习科学的活动设计
- 实时课堂互动与协作

**Check（过程性评估）**
- 多维度数据采集（行为、认知、情感）
- 学习轨迹可视化分析
- 形成性评估与诊断性评估

**Act（循证改进）**
- 学习分析技术生成洞察
- 数据驱动的教学决策
- 持续优化与迭代改进

#### 🌟 5E科学学习活动循环（学生视角）

```
┌─────────────────────────────────────────────────┐
│         5E 科学学习活动循环                      │
├─────────────────────────────────────────────────┤
│                                                  │
│  🎯 Engage (参与投入)                            │
│     ↓                                           │
│  🔍 Explore (探索发现)                           │
│     ↓                                           │
│  💡 Explain (解释建构)                           │
│     ↓                                           │
│  🚀 Elaborate (深化拓展)                         │
│     ↓                                           │
│  ✨ Evaluate (评价反思)                          │
│     ↑                                           │
│     └───────────────────────────────────────────┘
│                                                  │
│  基于建构主义、探究式学习、元认知理论            │
└─────────────────────────────────────────────────┘
```

**Engage（参与投入）**
- 激发好奇心，引发认知冲突
- 建立学习动机与问题意识
- 连接已有知识与新知识

**Explore（探索发现）**
- 动手实验操作，收集观察数据
- 形成初步概念理解
- 培养科学探究能力

**Explain（解释建构）**
- 表达个人理解，建构科学概念
- 形成系统化解释
- 发展科学语言能力

**Elaborate（深化拓展）**
- 应用新学知识，实现概念迁移
- 解决实际问题
- 发展高阶思维能力

**Evaluate（评价反思）**
- 评估学习成果，反思认知过程
- 发展元认知能力
- 形成自我调节学习习惯

> **双循环协同机制**：教学管理循环指导和优化学习活动循环，学习活动循环为教学管理循环提供数据反馈，形成持续改进的闭环。

## ✨ 核心功能亮点

### 1. 模块化教学设计系统

#### Cell 类型体系（7种核心类型）

| Cell类型 | 功能描述 | 应用场景 | 学习科学依据 |
|---------|---------|---------|-------------|
| **Text** | 富文本内容 | 知识讲解、概念介绍 | 多媒体学习理论 |
| **Code** | 可执行代码 | Python编程实践 | 做中学理论 |
| **Param** | 参数调节 | 变量控制实验 | 科学探究方法 |
| **Sim** | 仿真模拟 | PhET交互式实验 | 具身认知理论 |
| **QA** | 智能问答 | 苏格拉底式对话 | 元认知理论 |
| **Chart** | 数据可视化 | 数据分析与展示 | 视觉化学习 |
| **Contest** | 挑战任务 | 项目式学习 | 掌握学习理论 |

#### 特色功能
- **拖拽式编辑**：直观的模块化教学设计界面
- **实时预览**：全屏预览功能，以学生视角查看教学内容
- **模板库**：丰富的教学模板，快速创建优质课程
- **协作编辑**：支持多人协作设计课程（规划中）

### 2. 智能课堂管理系统

#### 实时课堂互动
- **课堂会话管理**：创建、加入、管理课堂会话
- **实时数据同步**：WebSocket实时通信，确保数据一致性
- **学生状态监控**：实时查看学生学习进度和参与情况
- **活动控制面板**：教师端统一控制课堂活动

#### 形成性评估
- **实时提交系统**：学生可实时提交作业和答案
- **即时反馈**：教师可即时查看和评价学生提交
- **统计分析**：自动生成班级学习数据统计
- **个性化指导**：基于数据提供个性化学习建议

### 3. 多模态学习资源

#### PhET 交互式仿真
- **100+仿真资源**：覆盖物理、化学、生物、数学等学科
- **无缝集成**：直接在课程中嵌入PhET仿真
- **参数可调**：支持自定义参数，适配不同教学场景

#### 代码执行环境
- **浏览器端执行**：基于JupyterLite的Python执行环境
- **服务端执行**：JupyterHub支持复杂计算任务

### 4. AI增强学习支持

## 🏗️ 技术架构

### 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                     前端层 (Frontend)                    │
│  Vue3 + TypeScript + Vite + TailwindCSS + Pinia        │
│  - 教师端：教案编辑器、课堂管理、数据分析                │
│  - 学生端：学习界面、作业提交、AI助手                    │
│  - 研究员端：数据可视化、研究报告                        │
└─────────────────────────────────────────────────────────┘
                            ↕ HTTP/WebSocket
┌─────────────────────────────────────────────────────────┐
│                     后端层 (Backend)                      │
│  FastAPI + Python 3.10+ + SQLAlchemy + Alembic         │
│  - RESTful API：课程、用户、活动、评估等                 │
│  - WebSocket：实时通信、课堂同步                          │
│  - AI服务：LangChain + DeepSeek + FAISS                 │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                   数据存储层 (Storage)                     │
│  - PostgreSQL + TimescaleDB：主数据库                     │
│  - Redis：缓存、会话管理                                  │
│  - MinIO：对象存储（图片、视频、文档）                    │
│  - ClickHouse：日志分析（规划中）                         │
└─────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────┐
│                   计算服务层 (Compute)                     │
│  - JupyterHub：服务端代码执行                             │
│  - JupyterLite：浏览器端Python执行                       │
│  - Kafka：消息队列（规划中）                              │
└─────────────────────────────────────────────────────────┘
```

### 技术栈详解

#### 前端技术栈
- **框架**：Vue 3.5+ (Composition API)
- **语言**：TypeScript 5.9+
- **构建工具**：Vite 7+
- **UI框架**：TailwindCSS 3.4+ + Shadcn/UI
- **状态管理**：Pinia 2.1+
- **路由**：Vue Router 4.2+
- **富文本编辑**：TipTap 2.1+
- **代码编辑**：CodeMirror 6
- **图表可视化**：ECharts 5.4+、Chart.js 4.4+
- **流程图**：AntV X6 2.18+
- **3D可视化**：Three.js 0.160+

#### 后端技术栈
- **框架**：FastAPI 0.104+
- **语言**：Python 3.10+
- **ORM**：SQLAlchemy 2.0+ (异步)
- **数据库迁移**：Alembic 1.12+
- **认证**：JWT (python-jose)
- **密码加密**：bcrypt (passlib)
- **文件处理**：Pillow、PyPDF2、python-docx
- **数据处理**：Pandas 2.1+

#### AI技术栈
- **框架**：LangChain
- **大语言模型**：DeepSeek
- **向量检索**：FAISS
- **上下文管理**：基于课程内容的RAG（检索增强生成）

#### 基础设施
- **容器化**：Docker + Docker Compose
- **数据库**：PostgreSQL 14+ (TimescaleDB扩展)
- **缓存**：Redis 7+
- **对象存储**：MinIO
- **消息队列**：Kafka (规划中)
- **时序数据库**：ClickHouse (规划中)

### 架构优势

1. **前后端分离**：清晰的职责划分，便于独立开发和部署
2. **异步处理**：全面采用异步编程，提升系统性能
3. **微服务化**：模块化设计，易于扩展和维护
4. **容器化部署**：Docker化部署，环境一致性保证
5. **可扩展性**：支持水平扩展，应对高并发场景

## 🎓 应用场景

### 1. K-12 STEM教育

#### 场景一：物理实验教学
- **问题**：传统物理实验受限于设备、安全、成本等因素
- **解决方案**：使用PhET仿真进行虚拟实验，学生可反复尝试，观察现象
- **效果**：提升实验安全性，降低教学成本，增强学习效果

#### 场景二：编程入门教学
- **问题**：编程教学需要环境配置，学生基础差异大
- **解决方案**：浏览器端代码执行，无需配置环境，支持个性化学习路径
- **效果**：降低学习门槛，提升学习兴趣，实现差异化教学

### 2. 高等教育

#### 场景一：项目式学习
- **问题**：项目式学习需要跨学科知识整合，管理复杂
- **解决方案**：模块化教学设计，支持多学科融合，实时跟踪项目进度
- **效果**：提升项目完成率，增强学生综合能力

#### 场景二：研究性学习
- **问题**：研究性学习缺乏数据支持和过程指导
- **解决方案**：学习分析技术提供数据支持，AI助手提供个性化指导
- **效果**：提升研究质量，培养科研能力

### 3. 教师专业发展

#### 场景一：教学设计优化
- **问题**：教师缺乏系统化的教学设计工具和方法
- **解决方案**：基于5E模型和布鲁姆分类法的教学设计工具
- **效果**：提升教学设计质量，节省备课时间

#### 场景二：教学效果评估
- **问题**：传统评估方式难以全面反映教学效果
- **解决方案**：多维度数据采集和分析，生成教学洞察报告
- **效果**：科学评估教学效果，支持持续改进

### 4. 教研活动

#### 场景一：课程资源共建
- **问题**：优质课程资源分散，缺乏共享机制
- **解决方案**：课程资源库，支持资源共享和协作编辑
- **效果**：提升资源利用效率，促进优质资源共享

#### 场景二：教学研究
- **问题**：教学研究缺乏数据支持
- **解决方案**：学习分析数据，支持教学研究
- **效果**：提升研究质量，推动教学创新

## 🚀 快速开始

### 系统要求

- **操作系统**：Windows 10+、macOS 10.15+、Linux (Ubuntu 20.04+)
- **Node.js**：18.0+
- **Python**：3.10+
- **Docker**：20.10+
- **Docker Compose**：2.0+
- **pnpm**：8.0+

### 一键启动

```bash
# 克隆项目
git clone https://github.com/your-org/inspireed-platform.git
cd inspireed-platform

# 使用一键启动脚本（推荐）
# Linux/macOS:
./start.sh

# Windows:
start.bat
```

一键启动脚本会自动完成：
- ✅ 启动 Docker 基础服务（PostgreSQL、Redis、MinIO等）
- ✅ 创建 Python 虚拟环境并安装依赖
- ✅ 运行数据库迁移
- ✅ 启动后端服务（FastAPI）
- ✅ 启动前端服务（Vite）
- ✅ 自动检测并显示局域网访问地址
- ✅ 显示测试账号信息

### 手动启动步骤

如果需要手动启动，请按照以下步骤操作：

#### 1. 启动基础服务

```bash
cd docker
docker-compose up -d
```

这将启动：
- PostgreSQL (端口 5432)
- Redis (端口 6379)
- MinIO (端口 9000/9001)
- Kafka + Zookeeper (端口 9092)

#### 2. 启动后端服务

```bash
# 进入后端目录
cd backend

# 创建虚拟环境（如果不存在）
python -m venv venv

# 激活虚拟环境
# macOS/Linux:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 创建环境配置文件
cp env.example .env

# 运行数据库迁移
alembic upgrade head

# 启动后端服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端将运行在：
- API地址：http://localhost:8000
- API文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

#### 3. 启动前端服务

```bash
# 进入前端目录
cd frontend

# 安装依赖
pnpm install

# 创建环境配置文件
cp env.example .env.local

# 启动前端开发服务器
pnpm dev
```

前端将运行在：http://localhost:5173

#### 4. 验证服务状态

```bash
# 检查后端健康状态
curl http://localhost:8000/health

# 检查前端服务
curl http://localhost:5173
```

### 局域网访问配置

系统已实现IP地址自动适配，支持局域网访问：

**自动适配（零配置）**
- ✅ 前端自动检测访问地址并匹配后端API
- ✅ 后端智能CORS，自动允许局域网IP段访问
- ✅ 启动服务后可直接使用任意局域网IP访问

**使用方法**：
1. 启动服务后，查看控制台输出的局域网访问地址
2. 从同一局域网内的其他设备（手机、平板、其他电脑）访问该地址
3. 无需额外配置，系统会自动适配

**手动配置**（可选）：
- 后端CORS配置：`backend/.env` 中的 `BACKEND_CORS_ORIGINS`
- 前端API地址：`frontend/.env.local` 中的 `VITE_API_BASE_URL`

详细配置说明请参考：[网络访问指南](docs/network/NETWORK_ACCESS_GUIDE.md)

### 常见问题

#### 后端启动问题

**问题：bcrypt 版本兼容性错误**
```bash
pip install "bcrypt<4.0.0" --force-reinstall
```

**问题：数据库连接失败**
```bash
# 检查 Docker 服务状态
docker-compose ps

# 重启数据库服务
docker-compose restart postgres
```

**问题：CORS 配置错误**
- 检查 `backend/.env` 中的 `BACKEND_CORS_ORIGINS` 配置
- 应该是 JSON 格式：`["http://localhost:5173","http://localhost:3000"]`

#### 前端启动问题

**问题：依赖安装失败**
```bash
# 清理缓存重新安装
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

**问题：端口被占用**
```bash
# 查找占用端口的进程
lsof -i :5173  # macOS/Linux
netstat -ano | findstr :5173  # Windows

# 杀死进程或使用其他端口
pnpm dev --port 3000
```

### 停止服务

```bash
# 停止所有 Docker 服务
cd docker && docker-compose down

# 停止后端服务
pkill -f "uvicorn app.main:app"

# 停止前端服务
pkill -f "pnpm dev"

# 或使用停止脚本
./stop.sh  # Linux/macOS
stop.bat   # Windows
```
